Service générique : createCustomerFreeAttribute


Ce service permet de créer une donnée complémentaire pour un client, correspondant aux critères de sélection contenus dans le flux XML qu'il reçoit après avoir vérifié leur cohérence.

/!\ Il est interdit de modifier une fiche client en attente de validation.

Nom : additionalData
Description : Donnée complémentaire du client.
Commentaire :

L'identifiant de la donnée complémentaire doit :

  • Exister en nomenclature 171,
  • Correspondre à une donnée complémentaire saisissable (Montant 2 différent de 2)
  • Etre autorisé pour le type de client (Libellé 4 valorisé à P ou S)
  • Etre unique pour le client

Valeur par défaut de l’identifiant : zone récupérée dans la table des valeurs par défaut pour l'identifiant de la donnée complémentaire et le type de client.

La valeur de la donnée complémentaire renseignée doit :

  • Correspondre au format prévu dans le Libellé 5,
  • Si son format est numérique, avoir un nombre de décimales inférieur ou égal au Montant 4,
  • Si son format est alphanumérique, avoir une longueur inférieure strictement au Montant 3, et exister dans la nomenclature dont le numéro est indiqué dans le Montant 6,
  • Si son format est de type date, être comprise entre la date calculée à partir de la date comptable moins le nombre d’années contenu dans le Montant 8 et la date calculée à partir de la date comptable plus le nombre d’années contenu dans le Montant 7.

Le type de la donnée complémentaire renseignée doit :

  • Correspondre au type prévu dans le Libellé 3

Le texte libre de la donnée complémentaire renseignée est autorisé si le Taux 3 n'est pas assigné à 0, et est obligatoire si le Taux 3 est assigné à 2.

La création des données complémentaires segment BHFM (SEG) et du flag prudentiel (FPR) n'est autorisée que si la gestion des segments et flag prudentiel BHFM est autorisée dans le Taux 1 (différent de 1) de la nomenclature 098 pour le paramètre BHFM. La valeur alphanumérique de ces données complémentaires doit exister dans la nomenclature 708.

Si le client appartient à un groupe, le flag prudentiel sera forcé à celui du groupe.

Sinon, si la gestion des données externes BHFM est activée dans le Taux 2 (à 1) de la nomenclature 098 pour le paramètre BHFM, alors la valeur du flag prudentiel sera alimentée avec le flag prudentiel de la maison mère si elle existe avec une date valide (antérieure à la date comptable) dans la table sghrefcli (pour les données externes RFPEMM et NUPERS). Si la maison mère existe mais n’a pas de flag prudentiel, le flag prudentiel du client est forcé à PME (P).

Liste des valeurs possibles pour la donnée complémentaire flag prudentiel :

  • R = Retail
  • P = PME
  • G = Grande entreprise

La création des données complémentaires liées à la cotation client FSC2 n'est autorisée que si le Libellé 3 de la nomenclature 098 pour le paramètre FSC2-SG existe en nomenclature 171 avec un Montant 2 égal à 2 (donnée complémentaire non saisissable et générée automatiquement) et un Libellé 5 égal à D (Date).

Ces données complémentaires sont :

  • La donnée complémentaire Date de cotation qui est paramétrée dans ce Libellé 3.
  • La donnée complémentaire Date de classement en douteux si elle est paramétrée dans le Libellé 1 de la nomenclature 098 pour le paramètre FSC2GE2-SG et qui doit exister en nomenclature 171 avec le Libellé 5 égal à D.
  • La donnée complémentaire Date de classement en douteux compromis si elle est paramétrée dans le Libellé 5 de la nomenclature 098 pour le paramètre FSC2GE2-SG et qui doit exister en nomenclature 171 avec le Libellé 5 égal à D.

Si le service de création d'une donnée complémentaire d'un client est appelé par le service modifyCustomer et qu'il y a changement de code qualité, alors la valeur de ces données complémentaires est modifiée avec la date comptable.
