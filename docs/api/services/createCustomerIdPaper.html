Programme Amplitude : cbwclipidc
Service générique : createCustomerIdPaper


Ce service permet de créer une pièce d'identité associée à un client correspondant aux critères de sélection contenus dans le flux XML qu'il reçoit après avoir vérifié leur cohérence.

/!\ Il est interdit de modifier une fiche client en attente de validation.

 

Concernant le traitement fonctionnel et les règles de gestion, se référer aux documents utilisateurs :

0020-3-11.XGestion bancaire - Référentiel - TOME 2

Nom : createCustomerIdPaperRequest
Description : Requête de création d'une pièce d'identité.
Commentaire :

Si le contrôle de liste noire est demandé dans le Taux 2 (égal à 1) dans le paramètre BLACKLISTE en nomenclature 098, alors le numéro de pièce d’identité, le type de pièce d’identité et la date de naissance du client sont recherchés dans le fichier de blacklist (dont la référence est paramétrée dans le Montant 4 du paramètre BLACKLISTE en nomenclature 098, ou dans le cas d’un contrôle dans la liste noire nationale uniquement, dans le Libellé 5 du paramètre BLACKLNAT en nomenclature 098).

Si un enregistrement du fichier de la blacklist est trouvé pour l’une des données contrôlées, alors les informations seront tracées dans la table des historiques de la blacklist.

Une opposition client est également créée, avec le code opposition paramétré dans le Montant 5 du paramètre BLACKLISTE en nomenclature 098, ou paramétré dans le Montant 5 du paramètre BLACKLNAT en nomenclature 098 si le contrôle est prévu sur la liste noire nationale, ce dernier étant précisé dans le Taux 5 (égal à 1) du paramètre BLACKLISTE en nomenclature 098.

Nom : createCustomerIdPaperRequest
Description : Requête de création d'une pièce d'identité.
Nom : customerCode
Description : Code du client.
Commentaire :

Le code client n'est pas obligatoire si le service de création d’une pièce d’identité d’un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

S’il est renseigné, il doit correspondre à un client existant dans la table des clients et doit être de type Particulier (1).

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Le client doit posséder une pièce d’identité principale rattachée à sa fiche pour pouvoir en définir une secondaire.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : idPaperType
Description : Type de pièce d'identité.
Commentaire :

Si le type de pièce d'identité est renseigné, alors le libellé du type de pièce d’identité est alimenté avec le Libellé 1 du type de pièce d’identité dans la nomenclature 078.

Il ne doit pas exister plus d’une pièce d’identité par type pour le client.
