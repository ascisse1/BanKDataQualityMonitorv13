Nom : dosprtImplementationFundsReleaseMethodInformation
Description : Informations sur la méthode de déblocage des fonds à la mise en place d'un dossier de crédit.
Choix : Non
  • dosprtImplementationFundsReleaseMethodInformation
    • paymentMethod : dosprtDefaultFundsReleasePaymentMethod
    • [C] paymentInformation : dosprtFundsReleasePaymentInformation
      • transferDomiciliation : accountIdentifierOtherBank
        • internalFormatAccount : internalFormatAccountOtherBank
          • establishment : charMax10
          • cashDesk : char5
          • accountNumber : charMax25
        • ibanFormatAccount : ibanFormatAccount
          • value : charMax34
      • otherAccount : internalAccountKey
        • branch : char5
        • currency : currencyCode
        • account : charMax11
        • suffix : charMax2

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : paymentMethod
Description : Mode de règlement.
Commentaire :

La méthode de déblocage des fonds n’est autorisée que si le dossier de prêt ne concerne pas un logement social.

La méthode de déblocage des fonds par remboursement n’est autorisée que si le dossier de crédit amortissable a un remboursement des échéances par remboursement.

Si le mode de règlement n’est pas renseigné, alors il est alimenté avec :

  • Autre compte (A) si le dossier de prêt est un dossier de prêt pour logement social
  • Le Libellé 2 du paramètre CDTMODREGL en nomenclature 098, ou par défaut avec Espèces (E), sinon.

L’étiquette comptable correspondante doit être paramétrée en nomenclature 094 pour la clé :

  • ESPPRETttt (avec ttt = type de prêt) si le mode de règlement est Espèces (E),
  • VRTPRETttt (avec ttt = type de prêt) si le mode de règlement est Virement (V),
  • CHQPRETttt (avec ttt = type de prêt) si le mode de règlement est Chèque (C).

Nom : otherAccount
Description : Autre compte.
Commentaire :

Les informations d’un autre compte ne sont prises en compte que pour un mode de règlement Autre compte (A). Elles sont interdites le cas échéant.

Le compte renseigné :

  • Doit exister,
  • Ne doit pas être fermé si le blocage sur compte fermé est paramétré (Montant 1 égal à 1 pour le paramètre DESACCORDS en nomenclature 098),
  • Ne doit pas être en instance de fermeture si le blocage sur compte en instance de fermeture est paramétré (Montant 2 égal à 1 pour le paramètre DESACCORDS en nomenclature 098),
  • Ne doit pas être déjà rattaché au dossier de prêt.

Dans le cas d’un dossier pour logement social, sont renseignés par défaut (et donc non saisissables) :

  • L’agence avec l’agence du dossier,
  • La devise avec la devise du dossier,
  • Le numéro de compte et le suffixe à partir du  Libellé 5 du paramètre PSCPENGttt (avec ttt = type de prêt) en nomenclature 094 (ou  PSCPENGPRT par défaut).
