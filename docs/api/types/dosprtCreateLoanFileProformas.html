Nom : loanFileProforma
Description : Pro-format.
Limite du nombre d'occurences mini-maxi : 1 aucune
Commentaire :

Si le paramètre CDTLOGIC1 existe en nomenclature 098 et que le Libellé 2 correspond à l’utilisateur, alors au moins un déblocage de fonds doit s’effectuer sur un compte fournisseur dont le type est autre que compte courant du client.

La somme des montants des pro-formats doit être égale au montant du prêt.

Description : Type de compte.
Nom : proformaAccountType
Description : Type de compte pour un pro-format.
Format : string
Valeurs possibles :
E Compte externe (Format NBS)
F Autre compte de la banque
C Compte courant du client
Nom : supplierCode
Description : Fournisseur.
Commentaire :

Si le type de compte n’est pas le compte courant (C) du client, alors le code fournisseur est obligatoire. Dans ce cas,

  • Il doit exister dans la table des fournisseurs,
  • La devise du fournisseur doit être la même que celle du dossier,
  • Si le type de compte est Compte externe (E), alors le code de l’établissement du fournisseur ne doit pas être identique à celui de la banque,
  • Si le type de compte est Compte externe (E) ou Autre compte de la banque  (F), alors le compte du fournisseur ne doit pas être renseigné dans la table des fournisseurs.

Nom : downPaymentAlreadyPaid
Description : Acompte déjà payé.
Commentaire :

Si le type de compte correspond au compte courant du client (C), alors l’acompte déjà payé ne doit pas être renseigné.

Sinon, il doit  être :

  • obligatoirement renseigné,
  • supérieur ou égal à 0,
  • inférieur au montant de pro-format.

Nom : downPaymentToBePaid
Description : Acompte à payer.
Commentaire :

Si le type de compte correspond au compte courant du client (C), alors l’acompte à payer ne doit pas être renseigné.

Nom : downPaymentMinimumAmountOverride
Description : Forçage du montant minimum d'acompte.
Commentaire :

Si le forçage du montant minimum d’acompte n’est pas voulu (FALSE) alors le montant de l’acompte total (déjà payé et restant à payer) doit être strictement inférieur au montant minimum paramétré pour la condition de nature MINPRF.

Le forçage du montant minimum d’acompte n’est pris en compte que si le niveau de forçage de l’utilisateur est supérieur ou égal au niveau de forçage paramétré dans le Montant 1 du paramètre PRFSAI en nomenclature 099.

Nom : boolean
