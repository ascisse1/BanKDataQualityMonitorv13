  • customerFreeAttributeInfo
Nom : customerFreeAttributeInfo
Description : Attribut libre d'un client.

Les données complémentaires des clients sont définies dans la nomenclature 171.

Choix : Non
  • customerFreeAttributeInfo
    • code : charMax10
    • designation : charMax30
    • dataFormat : freeAttributeFormat
    • totalLengthIfAlpha : integer
    • numberOfDecimalsIfAmountOrRate : short
    • managementMethod : additionalDataManagementMethod

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : code
Description : Code de l¿identifiant de donnée complémentaire.
Nom : charMax10
Description : Chaîne composée d'au plus 10 caractères.
Format : string
Longueur mini-maxi : 1 - 10
Nom : designation
Description : Désignation de la donnée complémentaire.
Nom : charMax30
Description : Chaîne composée d'au plus 30 caractères.
Format : string
Longueur mini-maxi : 1 - 30
Nom : dataFormat
Description : Format de la donnée.
Nom : freeAttributeFormat
Description : Format de la donnée complémentaire.
Format : string
Valeurs possibles :
M Montant
D Date
T Taux
A Alphanumérique
Nom : totalLengthIfAlpha
Description : Longueur totale de la donnée.
Commentaire :

Si la donnée est de format alphanumérique, alors sa longueur totale doit respecter la valeur inscrite dans le Montant 3 de la nomenclature 171 pour la donnée complémentaire considérée.

Nom : integer
Nom : numberOfDecimalsIfAmountOrRate
Description : Nombre de décimales de la donnée.
Commentaire :

Si la donnée est de format montant ou taux, alors son nombre de décimales doit respecter la valeur inscrite dans le Montant 4 de la nomenclature 171 pour la donnée complémentaire considérée.

Nom : short
Nom : managementMethod
Description : Mode de gestion de la donnée complémentaire.
Commentaire :

Le mode de gestion de la donnée complémentaire est saisissable (2) si le Montant 2 est égal à 2 dans la nomenclature 171 ; il est automatique (1) pour toute autre valeur du Montant 2.

Nom : additionalDataManagementMethod
Description : Mode de gestion.
Format : string
Valeurs possibles :
0 Cette donnée complémentaire est alimentée automatiquement et non affichée à l'utilisateur
1 Cette donnée complémentaire est saisissable par l'utilisateur
2 Cette donnée complémentaire est alimentée automatiquement
