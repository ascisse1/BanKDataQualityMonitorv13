Nom : branchCode
Description : Code agence.
Commentaire :

L'agence doit être différente du site central.
L'agence doit être autorisée pour l'utilisateur et ouverte.
 

Nom : branchCode
Description : Code d'une agence.
Format : string
Longueur fixe : 5
Nom : accountNumber
Description : Numéro de compte.
Commentaire :

Le numéro du compte doit être numérique et sa longueur doit être égale au Montant 1 du paramètre COMPTE dans la nomenclature 999.
Le compte doit exister dans la table des comptes et ne doit pas être en instance de fermeture ni fermé.
La devise du compte doit exister dans la nomenclature 005.
Le compte doit désigner un compte chèque.
Le client détenteur du compte doit exister dans la table des clients.
L'utilisateur doit être habilité sur le chapitre du compte et sur le profil du client. Le chapitre du compte doit également être autorisé pour l’opération.
Les conditions DATVAL et EXOCOM doivent exister pour ce compte.
 

Nom : accountNumber
Description : Numéro de compte.
Format : string
Longueur mini-maxi : 1 - 11
Nom : accountSuffix
Description : Suffixe du compte.
Commentaire :

Le suffixe est obligatoire si le Libellé 5 du paramètre COMPTE dans la nomenclature 999 est égal à O.
S'il est renseigné, le suffixe doit être numérique.
 

Nom : accountSuffix

Nom : stoppageApplicationDate
Description : Date d'application de l'opposition.
Commentaire :

La date d'application de la mise en opposition est modifiable uniquement si le paramètre MDOPCH existe dans la nomenclature 099 et que le niveau de forçage de l'utilisateur est supérieur ou égal au Montant 1.
Si elle est renseignée, la date doit être supérieure ou égale à la date comptable.
 

Nom : date

Nom : chequeNumber
Description : Numéro de chèque.
Commentaire :

La longueur du numéro du chèque doit être égale au Montant 3 du paramètre CHEQUES dans la nomenclature 098.
Si le chèque est en stock, alors :

  • ¿Il ne doit pas faire l'objet d’un autre évènement.
  • ¿Il doit appartenir au compte saisi.
  • ¿Il ne doit pas faire déjà l'objet d'une opposition.
  • ¿Il doit être remis.

Si le chèque n'est pas en stock, alors un enregistrement est créé dans la table des chèques.
