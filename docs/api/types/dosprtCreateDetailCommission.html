Nom : dosprtCreateDetailCommission
Description : Commission.

Si les commissions peuvent être calculées sur le plus fort solde débiteur du mois précédent sur le compte d'engagement du client du dossier de crédit (si le   Montant 6 du paramètre CDTPAR2  en nomenclature 098 est égal à 1) et que leur assurance est de type HDB (H) alors leur mode doit être proportionnel (P) dans la table des accessoires du dossier de prêt.

Choix : Non
  • dosprtCreateDetailCommission
    • [C] commissionAmountOrRate : amountOrRate
      • amount : decimal19_4
      • rate : decimal8_6
    • commissionMaturity : dosprtDetailMaturityType
    • commissionPeriodicity : dosprtDetailPeriodicityType
    • commissionCollectionMethod : dosprtDetailCollectionMethodType

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : commissionAmountOrRate
Description : Montant ou taux de la commission.
Commentaire :

La balise renseignée (Taux ou Montant) doit être en accord avec le paramétrage (M ou T) de la commission  dans les conditions COxttt ou COxPRT où ttt correspond au code du type de prêt, et x correspond à 1, 2 ou 3 selon la commission 1, 2 ou 3.

Le montant ou le taux de la commission x doit être compris entre la valeur minimum de la condition MICxPR et la valeur maximum de la condition MACxPR où x correspond à 1, 2 ou 3 selon la commission 1, 2 ou 3.

Nom : amountOrRate
Description : Montant ou taux.
Nom : amount
Description : Montant.
Nom : decimal19_4
Description : Type décimal à 19 chiffres dont 4 décimales.
Format : decimal
Nombre total de chiffres : 19
Nom : rate
Description : Taux.
Nom : decimal8_6
Description : Type décimal à 8 chiffres dont 6 décimales.
Format : decimal
Nombre total de chiffres : 8
Nom : commissionMaturity
Description : Terme de la commission.
Commentaire :

Valeur par défaut selon la  commission 1, 2 ou 3 :

  • ter_co1
  • ter_co2
  • ter_co3
Nom : dosprtDetailMaturityType
Description : Terme d'un poste.
Format : string
Valeurs possibles :
A Perçu d'avance
E A l'échéance
Nom : commissionPeriodicity
Description : Périodicité de la commission.
Commentaire :

Valeur par défaut selon la  commission 1, 2 ou 3 :

  • per_co1
  • per_co2
  • per_co3

La périodicité de la  commission, si elle est renseignée, doit être un multiple du nombre de mois du prêt (durée du prêt calculée par différence entre la date de dernière et la date de première échéance, ou durée du prêt renseignée directement par le nombre d'échéances).

Nom : dosprtDetailPeriodicityType
Description : Périodicité.
Format : integer
Min-max inclusive : 1 - 12
Nom : commissionCollectionMethod

Commentaire :

Valeur par défaut selon la commission 1, 2 ou 3 :

  • mpr_co1
  • mpr_co2
  • mpr_co3
