  • cardChipApplication
Nom : cardChipApplication
Description : Application puce d'une carte.

Les limites des applications puces sont spécifiques aux types de carte du processeur monétique FIS et aux cartes à puce. Si cette balise est utilisée dans d'autre cas, un avertissement non bloquant indiquera qu'elle est ignorée.

Choix : Non
  • cardChipApplication
    • chipApplicationID : chipApplicationID
    • [M] chipApplicationExceptionalLimit : exceptionalCardLimit
      • cardExceptionalLimitCode : charMax10
      • cardExceptionalLimitValue : decimal19_4
      • cardExceptionalLimitStartDate : date
      • cardExceptionalLimitEndDate : date

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : chipApplicationID
Description : Identifiant de l'application puce.
Commentaire :

Doit exister dans la table des applications puces et être défini pour le type de la carte traitée.

Nom : chipApplicationID
Description : Identifiant application puce.
Format : string
Longueur mini-maxi : néant - 16
Nom : chipApplicationExceptionalLimit
Description : Limites exceptionnelles de l'application puce.
Limite du nombre d'occurences mini-maxi : 1 aucune
Nom : exceptionalCardLimit
Description : Informations sur un plafond exceptionnel.
Nom : cardExceptionalLimitCode
Description : Code du plafond exceptionnel.
Nom : charMax10
Description : Chaîne composée d'au plus 10 caractères.
Format : string
Longueur mini-maxi : 1 - 10
Nom : cardExceptionalLimitValue
Description : Valeur du plafond exceptionnel.
Nom : decimal19_4
Description : Type décimal à 19 chiffres dont 4 décimales.
Format : decimal
Nombre total de chiffres : 19
Nom : cardExceptionalLimitStartDate
Description : Date de début du plafond exceptionnel.
Nom : date
Nom : cardExceptionalLimitEndDate
Description : Date de fin du plafond exceptionnel.
