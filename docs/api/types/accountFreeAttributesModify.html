Nom : additionalData
Description : Donnée complémentaire d'un compte.
Commentaire :

L’identifiant de la donnée complémentaire d’un compte ne peut pas être celui paramétrée dans le Libellé 2 en nomenclature 098 pour CBRACCOUNT.

L’identifiant de la donnée complémentaire d’un compte doit :

  • exister en nomenclature 168,
  • correspondre à une donnée complémentaire saisissable (Montant 2 différent de 2),
  • être autorisé pour le type de compte de clientèle (Libellé 4 valorisé à C ou T).

La valeur de la donnée complémentaire renseignée doit :

  • correspondre au format prévu dans le Libellé 5,
  • si son format est numérique, avoir un nombre de décimales inférieur ou égal au Montant 4,
  • si son format est alphanumérique, avoir une longueur maximum au Montant 3, et exister dans la nomenclature dont le numéro est indiqué dans le Libellé 6.

Si le service est appelé par le service createAccount, une vérification de l’existence de toutes les données complémentaires obligatoires (Montant 1 renseigné à 1 dans la nomenclature 168) et saisissables (Montant 2 renseigné à 1 dans la nomenclature 168) d’un compte est effectuée.

Valeur par défaut : identifiant de la donnée complémentaire (uniquement pour la création d’une donnée complémentaire).
