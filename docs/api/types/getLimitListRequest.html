  • getLimitListRequest
Nom : getLimitListRequest
Description : Requête pour la liste des limites d'un client ou groupe de clients.
Choix : Non
  • getLimitListRequest
    • [C] identifier : limitListIdentifier
      • customer : customerCode
      • customersGroup : customersGroupCode
    • limitLevelCode : limitLevelCode
    • limitStatus : limitStatus
    • beginningDate : date
    • endingDate : date
    • customerLimitIncluded : boolean
    • customerGroupLimitIncluded : boolean
    • globalLineIncluded : boolean
    • subLimitGlobalLineDetailIncluded : boolean

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : identifier
Description : Identifiant du client ou du groupe de clients.
Nom : limitListIdentifier
Description : liste des identifiants de limite.
Nom : customer
Description : Code client.
Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : customersGroup
Description : Code groupe de clients.
Nom : customersGroupCode
Description : Code groupe.
Format : string
Longueur mini-maxi : 1 - 6
Nom : limitLevelCode
Description : Niveau des limites à rechercher.
Nom : limitLevelCode
Description : Code niveau de la limite.
Format : string
Valeurs possibles :
MA limite de marché
GS limite globale avec saisie des sous-limites
GD limite globale déduite du détail des sous-limites
US limite universelle avec saisie des sous-limites
UN limite universelle
* toutes
Nom : limitStatus
Description : Situation des limites à rechercher
Nom : limitStatus
Description : Situation des limites à rechercher
Format : string
Valeurs possibles :
W attente validation modification
X attente validation annulation
C créée
T attente validation création
M modifiée
* toute

Nom : date
Nom : endingDate
Description : Date d'échéance maximale des limites à rechercher.

Nom : customerGroupLimitIncluded
