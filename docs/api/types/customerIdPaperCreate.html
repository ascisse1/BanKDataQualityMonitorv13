Nom : customerIdPaperCreate
Description : Informations d¿une pièce d¿identité pour un particulier.

Pour un client de type Particulier (1).

Si le contrôle de liste noire est demandé dans le Taux 2 (égal à 1) du paramètre BLACKLISTE en nomenclature 098, alors le numéro de pièce d’identité, le type de pièce d’identité et la date de naissance sont recherchés dans le fichier de blacklist (dont la référence est paramétrée dans le Montant 4 du paramètre BLACKLISTE en nomenclature 098, ou dans le cas d’un contrôle dans la liste noire nationale uniquement, dans le Libellé 5 du paramètre BLACKLNAT en nomenclature 098).

Si un enregistrement du fichier de la blacklist est trouvé pour l’une des données contrôlées, alors les informations seront tracées dans la table des historiques de la blacklist.

Une opposition client est également créée, avec le code opposition paramétré dans le Montant 5 du paramètre BLACKLISTE en nomenclature 098, ou paramétré dans le Montant 5 du paramètre BLACKLNAT en nomenclature 098 si le contrôle est prévu sur la liste noire nationale, ce dernier étant précisé dans le Taux 5 (égal à 1) du paramètre BLACKLISTE en nomenclature 098.

Choix : Non

  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Nom : type
Description : Type de pièce d'identité.
Commentaire :

Le type de pièce d’identité est obligatoire si le numéro de pièce d’identité est renseigné.

Valeur par défaut : tid

Nom : idPaperTypeCode
Description : Code du type de pièce d'identité.
Format : string
Longueur mini-maxi : 1 - 5
Nom : idPaperNumber
Description : Numéro de pièce d'identité.
Commentaire :

Le numéro de pièce d’identité est obligatoire si le lieu de délivrance ou la date de délivrance est renseigné.

Si pour le type de pièce d’identité, une longueur du numéro de pièce d’identité est paramétrée dans le Montant 4 de la nomenclature 078 des types de pièce d’identité, alors le numéro de pièce d’identité doit avoir cette longueur.

Si le contrôle d’unicité du numéro de pièce d’identité est demandé dans le Libellé 2 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 1)) du paramètre CTRIDENCLI en nomenclature 098, alors :

  • il ne doit pas exister de client avec les mêmes numéro de pièce d’identité, type de pièce d'identité et code intitulé
  • si le client est un particulier de nature non collectif (2), alors il ne doit pas exister de client avec les mêmes numéro et type de pièce d’identité et pour lequel le code intitulé ne serait pas associé à un client particulier de nature collectif (nature paramétrée dans le Montant 2 dans la nomenclature 036).

Valeur par défaut : nid

Nom : charMax20
Description : Chaîne composée d'au plus 20 caractères.

Longueur mini-maxi : 1 - 20
Nom : idPaperDeliveryDate
Description : Date de délivrance de la pièce.
Commentaire :

La date de délivrance de la pièce d’identité ne peut être renseignée que si le numéro de pièce d’identité l’est.

Si la date et le lieu de délivrance de la pièce sont renseignés, alors la date ne doit pas être :

  • Postérieure à la date comptable
  • Antérieure à la date de naissance du client

Valeur par défaut (contrôle de l’obligation de saisie uniquement) : did

Il doit être renseigné si l’organisme ayant délivré la pièce est renseigné.

Valeur par défaut : oid

Nom : charMax30
Description : Chaîne composée d'au plus 30 caractères.
Format : string
Longueur mini-maxi : 1 - 30
Nom : idPaperValidityDate
Description : Date de validité de la pièce.
Commentaire :

La date de validité de la pièce d’identité ne peut être renseignée que si le numéro de pièce d’identité l’est.

Si ces informations sont renseignées, alors la date de validité de la pièce doit être supérieure à sa date de délivrance. La date de validité doit également être supérieure à la date comptable.

Valeur par défaut (contrôle de l’obligation de saisie uniquement) : vid

Nom : date
Nom : nationalIdentifier
Description : Numéro d'identité nationale.
Commentaire :

Pour un particulier de nature non collectif (2), si le numéro d’identité nationale est renseigné alors :

  • Si le contrôle d’unicité est demandé sur le numéro d’identité nationale dans le Libellé 5 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 1)) du paramètre CTRIDENCL2 en nomenclature 098, alors le numéro d’identité nationale ne doit pas déjà exister dans la table des clients.
  • Sinon, si le contrôle d’unicité est demandé sur le numéro d’identité et sur l’intitulé dans le Libellé 2 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 1)) mais pas sur le numéro d’identité nationale dans le Libellé 5 (différent de O et B) du paramètre CTRIDENCL2 en nomenclature 098, alors le numéro d’identité nationale associé au code intitulé du client ne doit pas déjà exister dans la table des clients
  • Sinon, si le contrôle d’unicité est demandé sur le numéro d’identité seul dans le Libellé 5 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 1)) du paramètre CTRIDENCL2 en nomenclature 098, alors le numéro d’identité nationale ne doit pas déjà exister dans la table des clients.

Valeur par défaut : nidn
