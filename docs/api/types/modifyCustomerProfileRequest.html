  • modifyCustomerProfileRequest
Nom : modifyCustomerProfileRequest
Description : Requête de modification du profil d'un client.

L'utilisateur effectuant la modification doit être habilité à affecter le profil donné au client.

Choix : Non
  • modifyCustomerProfileRequest
    • customerCode : customerCode
    • profileCode : customerProfileCode
    • applicationDate : date

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : customerCode
Description : Code client.
Commentaire :

Le code client doit correspondre à un client existant dans la table des clients.

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Le client doit toujours avoir un profil actif, c'est-à-dire un profil dont la date d’application est inférieure ou égale à la date comptable.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : profileCode
Description : Code profil.
Commentaire :

Le code profil doit exister en nomenclature 050. Si une valeur est renseignée dans le Montant 1, elle doit correspondre au type du client.

Valeur par défaut : pro

Nom : customerProfileCode
Description : Code profil d'un client.
Format : string
Longueur mini-maxi : 1 - 3
Nom : applicationDate
Description : Date d¿application.
Commentaire :

La date d’application doit être supérieure ou égale à la date comptable.

Valeur par défaut : dpro
