Nom : customerOrThirdPartyCoHolder
Description : Co-titulaire client ou tiers.
Choix : Non
  • customerOrThirdPartyCoHolder
    • coHolderType : customerOrThirdPartyType
    • jointCustomerCode : customerCode

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : coHolderType
Description : Type de co-titulaire.
Nom : customerOrThirdPartyType
Description : Population ciblée.
Format : string
Valeurs possibles :
C Client
T Tiers
Nom : jointCustomerCode
Description : Code du client joint.
Commentaire :

Le code du client joint doit être différent du code du client principal.

Si le type de co-titulaire désigne un client (C), alors le code du client joint doit identifier un enregistrement dans la table des clients.

Sinon, si le type de co-titulaire désigne un tiers (T), alors le code du client joint doit correspondre à un enregistrement dans la table des tiers.

Si le paramètre COOP n’est pas présent en nomenclature 098, le client ou le tiers ne doit pas être un collectif, c'est-à-dire que le Montant 2 de la nomenclature 036 lue sur son code intitulé ne doit pas être valorisé à 2.

Si le paramètre COOP n’est pas présent en nomenclature 098, et si le Montant 2 pour le paramètre CTRLDIVCLI en nomenclature 098 est différent de 1, alors le co-titulaire doit être un particulier.

Si le paramètre COOP est présent en nomenclature 098, le nom du co-titulaire est formaté avec la concaténation du code client et du nom, séparés par un espace.

Un couple "type de co-titulaire/code du client joint" ne doit figurer qu’une seule fois dans le tableau des comptes joints à lier au client.

Nom : customerCode
Description : Code client.
Format : string
