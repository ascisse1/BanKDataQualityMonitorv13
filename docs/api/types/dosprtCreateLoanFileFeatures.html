Nom : dosprtCreateLoanFileFeatures
Description : Caractéristiques du dossier de crédit.
Choix : Non

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : jointBorrowersNumber
Description : Nombre de co emprunteurs.
Commentaire :

Si le nombre de co¿emprunteurs n’est pas renseigné, alors il est placé à 1, sinon il doit être supérieur à 0.

De plus, si le code client n’est pas renseigné dans l’élément "customerCode" du type "dosprtCreateLoanFile" défini dans ce document, alors le nombre de co¿emprunteurs ne doit pas être supérieur à 1.

Nom : integer
Nom : letterOfGuaranteeNumber
Description : Nombre de cautions.
Commentaire :

Vaut par défaut 0.

Le nombre de cautions ne peut pas être supérieur à 0 si le code client n’est pas renseigné.

Nom : integer
Nom : financingSource
Description : Source de financement.
Commentaire :

Si une source de financement factice est paramétrée dans le Libellé 2 du paramètre CDTPAR5 en nomenclature 098, la source de financement reçue sera ignorée.

La source est obligatoire si au moins une étape de mise en place du dossier est à comptabiliser (comptabilité étude, comptabilité banque, comptabilité client ou comptabilité à la validation finale).

La source de financement doit exister en nomenclature 123. 

Si une étape de mise en place est à comptabilisation étude, le compte paramétré dans le Libellé 5 pour la source de financement en  nomenclature 123 doit exister.

Si une étape de mise en place est à comptabilisation banque, client ou validation finale et n’est pas à comptabilisation étude, le compte paramétré dans le Libellé 4 pour la source de financement en  nomenclature 123 doit exister. Dans ce cas également, si une source factice est paramétrée,  les 2 comptes paramétrés dans les Libellé 4 et Libellé 5  pour la source de financement en nomenclature 123 doivent être identiques.

Dans ces cas et si le code client du dossier est renseigné, alors un événement CRSOUR est ajouté à la table des événements.

Lors de la modification d’un dossier, si le dossier est à comptabiliser et si le code client du dossier est renseigné alors qu'il ne l'était pas précédemment (ou si la source de financement est différente de celle déjà associée au prêt dans le cas d’une modification) alors un événement CRSOUR est ajouté à la table des événements

Valeur par défaut : source pour le type de prêt.

Nom : dosprtFinancingSourceCode
Description : Code de la source de financement d'un crédit.
Format : string
Longueur fixe : 4
Nom : correspondentMargin
Description : Marge du correspondant.
Commentaire :

La marge du correspondant ne doit être renseignée que si la source de financement est renseignée et que le Montant 2 est égal à 1 pour la source de financement en nomenclature 123.

Si la marge du correspondant n’est pas renseignée, elle sera affectée avec la valeur par défaut paramétrée dans la condition MCssss (ssss = source de financement).

Elle doit être comprise entre les valeurs minimum et maximum paramétrées dans la condition MCssss (ssss = source de financement) ou entre -999,999999 et +999,999999 si la condition n’est pas  paramétrée.

Elle doit être différente de 0.

Nom : decimal9_6
Description : Type décimal à 9 chiffres dont 6 décimales.
Format : decimal
Nombre total de chiffres : 9
Nom : personalInvestmentAmount
Description : Montant de l'apport personnel.
Commentaire :

Le montant d'apport ne doit pas être renseigné si le client ne l'est pas dans l’élément "customerCode" du type "dosprtCreateLoanFile" défini dans ce document.

De plus, il ne doit pas être renseigné si le mode de déblocage paramétré pour le type de dossier est différent de A ou F.

Nom : renegotiatedFile
Description : Dossier renégocié.
Commentaire :

Si cette donnée est à TRUE, les contrôles suivants sont effectués :

  • Le Libellé 4 du paramètre CDTPAR3 en nomenclature 098 doit être renseigné.
  • De plus, la valeur de ce libellé ne doit pas être différente du code qualité client.
Nom : boolean
Nom : originalFiles
Description : Dossier d'origine dans le cas d'un dossier renégocié.
Commentaire :

Les informations concernant le dossier d’origine ne peuvent être renseignées que si le dossier est signalé renégocié (TRUE).

Nom : dosprtOriginalFiles
Description : Gestions des dossiers d'origine dans le cas d'un dossier renégocié.
Nom : originalFile
Description : Identifiant d¿un crédit amortissable.
Limite du nombre d'occurences mini-maxi : 1 aucune
Commentaire :

L’identifiant du dossier doit exister dans la table des crédits amortissables.

Si le Taux 5 du paramètre CDTPAR6 en nomenclature 098 est égal à 1, alors le dossier de crédit amortissable d’origine doit être en attente de tombée, en attente de mise en place après une tombée, échu ou déchu de son terme.

Si le Taux 5 du paramètre CDTPAR6 en nomenclature 098 est différent de 1, alors le dossier de crédit amortissable d’origine doit être échu ou déchu de son terme.

Un identifiant de crédit amortissable ne peut être mentionné qu’une seule fois dans la liste des dossiers d’origine.
