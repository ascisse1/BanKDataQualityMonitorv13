Nom : dosprtCreateIndexedRate
Description : Taux d'intérêts indexés.
Choix : Non
  • dosprtCreateIndexedRate
    • baseRateCode : loanBaseRateCode
    • marginValue : decimal8_6

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : baseRateCode
Description : Code du taux de base.
Commentaire :

Le code du taux de base ne doit être renseigné que dans le cas d'un taux indexé et dans le cas où la gestion des codes de taux de base ne se fait pas par le biais de conditions (le Libellé 2 du paramètre CDTPAR2 doit être égal à O).

Le code du taux de base renseigné doit :

  • exister dans la table des taux de base,
  • être autorisé pour le type de prêt et la devise du prêt.

Si aucun code de taux de base n'est renseigné en entrée, alors le code du taux de base paramétré pour le type et la devise du prêt est affecté par défaut s’il est unique, sinon la saisie du taux de base est obligatoire.

Nom : loanBaseRateCode
Description : Code du taux de base.
Format : string
Longueur fixe : 3
Nom : marginValue
Description : Valeur de la marge.
Commentaire :

La valeur de la marge peut être renseignée uniquement si elle est saisissable (Libellé 2 du paramètre PRET en nomenclature 098 égal à O). Si le taux est indexé, alors le taux d’intérêts final est automatiquement recalculé en fonction de la marge renseignée et du taux de base recherché à partir du code de taux de base lu dans la condition TBssss (ssss = source de financement) ou à défaut TBPRET.

Si la valeur de la marge ne peut pas être renseignée (Libellé 2 du paramètre PRET en nomenclature 098 égal à N), alors elle est positionnée à la valeur de la condition VBssss (ssss = source de financement) ou à défaut INTPRI.

Nom : decimal8_6
Description : Type décimal à 8 chiffres dont 6 décimales.
Format : decimal
Nombre total de chiffres : 8
