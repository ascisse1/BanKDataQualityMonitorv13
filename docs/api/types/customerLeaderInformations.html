Nom : leaderCode
Description : Code du dirigeant.
Commentaire :

Si le type de dirigeant désigne un client (C), alors le code dirigeant doit identifier un enregistrement dans la table des clients.

Sinon, si le type de dirigeant désigne un tiers (T), alors le code dirigeant doit correspondre à un enregistrement dans la table des tiers.

Si le contrôle d’unicité du dirigeant est demandé dans le paramètre CONTROLDIR en nomenclature 098 avec le Libellé 2 renseigné à B, alors le couple nom/prénom du dirigeant ne doit pas déjà exister dans la table des dirigeants au moment de la création, ou ne doit pas être en doublon dans le flux entrée du service "modifyCustomerLeaders".

Dans le cas où le Libellé 2 est à O, un contrôle non bloquant est effectué, sinon, si le libellé est laissé à blanc aucun contrôle d’unicité n’est appliqué.

Valeur par défaut du nom du dirigeant : nomd

Valeur par défaut du prénom du dirigeant : pred

Valeur par défaut de la nationalité du dirigeant : natd

Nom : leaderType
Description : Type de dirigeant.
Commentaire :

Si le contrôle que le type de dirigeant ne doit pas être de type société est demandé dans le paramètre TYPFONDIR en nomenclature 098 avec le Libellé 4 différent de O, alors le client/tiers doit être de type Particulier.

Si le contrôle que le dirigeant ne doit être ni un client ni un tiers collectif est demandé dans le paramètre TYPFONDIR en nomenclature 098 avec le Libellé 5 renseigné à O, alors le code intitulé du client ou du tiers doit être présent en nomenclature 036 et son Montant 2 doit être de nature Client collectif (2).

Nom : leaderPosition
Description : Fonction du dirigeant.
Commentaire :

Il est interdit de renseigner à la fois la fonction du dirigeant et le type de fonction du dirigeant.

La fonction du dirigeant peut être alimentée selon le paramétragre de la clé TYPFONDIR en nomenclature 098 :

  • Si le type de fonction du dirigrant n’est pas géré, c'est-à-dire si le Libellé 2 vaut N,
  • Ou bien si la saisie libre de la fonction du dirigeant est autorisée, c'est-à-dire si le Libellé 3 vaut O.

Si les deux libellés sont valorisés à O alors une seule des deux données doit être renseignée.

Si le Libellé 3 interdit la saisie libre (valeur positionnée à N) alors la fonction du dirigeant est ignorée et seul le type de fonction est considéré.

Valeur par défaut : fdir

Nom : typeOfCorporateExecutiveDuty
Description : Type de fonction du dirigeant.
Commentaire :

Il est interdit de renseigner à la fois la fonction du dirigeant et le type de fonction du dirigeant.

Le type de fonction du dirigeant peut être alimenté si le type de fonction du dirigrant est géré, c'est-à-dire si le Libellé 2 du paramètre TYPFONDIR en nomenclature 098 vaut O.

Dans ce cas, le type de fonction du dirigeant doit exister en nomenclature 197.

Si le Libellé 2 et le Libellé 3 du paramètre TYPFONDIR en nomenclature 098 sont valorisés à O alors une seule des deux données doit être renseignée.

Si le Libellé 2 indique que le type de fonction n’est pas géré (valeur positionnée à N) alors le type de fonction du dirigeant est ignoré et seule la fonction en libre saisie est considérée.

Valeur par défaut : typfdir
