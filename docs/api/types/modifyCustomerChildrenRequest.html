Nom : modifyCustomerChildrenRequest
Description : Requête modification des enfants d'un client.
Choix : Non
  • modifyCustomerChildrenRequest

    • [M] customerChild : customerChildInformations
      • name : charMax36

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : customerCode
Description : Code du client.
Commentaire :

Le code client n’est pas obligatoire si le service de modification des enfants d’un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

S’il est renseigné, il doit correspondre à un client existant dans la table des clients et doit être de type Particulier (1).

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : customerChild
Description : Enfant du client.
Limite du nombre d'occurences mini-maxi : aucune aucune
Commentaire :

Le service "modifyCustomerChildren" procède par "annule et remplace". Il est donc nécessaire de passer la totalité des enfants à lier au client.

Le nombre maximum d’enfants autorisés pour un client est de 70.

Le nombre d’enfants reçus doit être inférieur ou égal au nombre d’enfants déclarés dans la table des clients.
