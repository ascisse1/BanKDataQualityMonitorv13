Nom : comparisonDate
Description : Comparaison de dates.
Choix : Non
  • comparisonDate
    • date1 : date
    • date2 : date
    • comparator : comparator

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : date1
Description : 1ère date.
Nom : date
Nom : date2
Description : 2ème date.
Commentaire :

La 2ème date n’est prise en compte que pour les opérateurs de comparaison BETWEEN et BETWEEN EQUALS.

N'est jamais renseignée dans un flux de réponse.

Nom : date
Nom : comparator
Description : Opérateur de comparaison.
Commentaire :

Vaut EQUALS dans un flux de réponse.

Nom : comparator
Description : Opérateur de comparaison.
Format : string
Valeurs possibles :
GREATER THAN Supérieur strict
NOT EQUALS Différent de
GREATER EQUALS Supérieur ou égal à
LOWER EQUALS Inférieur ou égal
BETWEEN Compris entre bornes exclues
BETWEEN EQUALS Compris entre bornes incluses
EQUALS Egal à
LOWER THAN Inférieur strict
