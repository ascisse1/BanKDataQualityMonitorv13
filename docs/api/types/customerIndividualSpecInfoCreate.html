Commentaire :

Si le contrôle spécifique du prénom est demandé dans le Montant 1 (égal à 1) du paramètre CTRLNOMCLI en nomenclature 098, alors le prénom ne doit contenir que des caractères alphabétiques.

Le prénom est modifiable si le niveau de forçage de l’utilisateur est supérieur ou égal au niveau de forçage paramétré dans le Montant 1 du paramètre MODNOM en nomenclature 099.

  • dirigeants correspondant au client dans la table des dirigeants,
  • actionnaires correspondant au client dans la table des actionnaires,
  • comptes joints du client dans la table des comptes joints.

Sinon, si le type de co-titulaire désigne un tiers (T), alors le code du client joint doit correspondre à un enregistrement dans la table des tiers.

Si le paramètre COOP n’est pas présent en nomenclature 098, le client ou le tiers ne doit pas être un collectif, c'est-à-dire que le Montant 2 de la nomenclature 036 lue sur son code intitulé ne doit pas être valorisé à 2.

Si le paramètre COOP n’est pas présent en nomenclature 098, et si le Montant 2 pour le paramètre CTRLDIVCLI en nomenclature 098 est différent de 1, alors le co-titulaire doit être un particulier.

Si le paramètre COOP est présent en nomenclature 098, le nom du co-titulaire est formaté avec la concaténation du code client et du nom, séparés par un espace.

Un couple "type de co-titulaire/code du client joint" ne doit figurer qu’une seule fois dans le tableau des comptes joints à lier au client.
