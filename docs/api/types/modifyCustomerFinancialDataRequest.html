Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : customerCode
Description : Code client.
Commentaire :

Le code client n’est pas obligatoire si le service de modification des données financières d’un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

S’il est renseigné, le code client doit correspondre à un client existant dans la table des clients.

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Le client ne doit pas être de type Particulier (1).

Nom : dataType
Description : Type de la donnée financière.
Commentaire :

Le type de la donnée financière doit exister en nomenclature 189.

  • Si le Montant 1 de la nomenclature 098 pour le paramètre VDEF-DCOMP est différent de 1 et le Taux 1 de la nomenclature 098 pour le paramètre DONFINANCE est égal à 1, alors :
    • Si l’entreprise a moins de 2 ans, aucun contrôle d’obligation n’est appliqué aux données financières.
    • Sinon (l’entreprise a au moins 2 ans), pour toutes les données financières dont les types sont paramétrés obligatoires (Libellé 3 renseigné à O dans la nomenclature 189), la valeur de la donnée financière correspondant à l’année courante ou à l’année courante -1 doit être fournie dans le flux d’entrée.
  • Si le Montant 1 de la nomenclature 098 pour le paramètre VDEF-DCOMP est différent de 1 et le Taux 1 de la nomenclature 098 pour le paramètre DONFINANCE est différent de 1, alors pour toutes les données financières dont les types sont paramétrés obligatoires (Libellé 3 renseigné à O dans la nomenclature 189), la valeur de la donnée financière de l’année courante doit être fournie dans le flux entrée.

Si le Montant 1 de la nomenclature 098 pour le paramètre VDEF-DCOMP est égal à 1, alors pour tous les types de données financières de la nomenclature 189 (qu’ils soient obligatoires ou non), toutes les données financières dont la valeur est paramétrée comme obligatoire dans la gestion des valeurs par défaut doivent être fournies dans le flux en entrée.
