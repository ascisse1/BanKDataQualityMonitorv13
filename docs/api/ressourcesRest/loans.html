Service SOAP : cancelAmortizableLoan
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/loans/v1/{loanId}
Description : The service cancelAmortizableLoan allows cancelling an instalment loan file corresponding to a unique identifier received. Only incomplete files under examination (status code SI and check code 0), complete files under examination (status code SI and check code 1), files accepted by the bank (status code VB and check code 1), and files accepted by the customer (status code VC and check code 1) can be cancelled. If management of the history of the instalment loan type is activated, then one cancellation line is inserted into the loan history table.

Global information about the loan file, description and incidental expenses are needed. They can be accompanied by other additional information: - Characteristics of loan file, - Credit object, - Documents, - Additional information, - Invoices, - Internal insurances, - Notes, - Instalment distribution between participants, - Repayment information, - Bank information of creditor¿s account, - Revaluation information, - Exchange rates used for conversions, - Marginal costs, - Pro forma, - Ancillary costs, - External charges, - Collateral information for EIR calculation, - Annual management commission, - Information about funds reservation for instalments, - Commitment commission. The service sends back the amortizable loan identifier created.

Service SOAP : getAmortizableLoanList
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/loans/v1/
Description : The service getAmortizableLoanList allows returning a list of instalment loan files matching the search criteria received. The search is based on the optional criteria below: - the loan file identifier: file number, order number, branch (code and/or designation), amendment number, - the currency (alphanumeric code and/or digital code and/or designation), - the customer: code and/or name to be displayed, type, profile (code and/or designation), national identifier, account officer (code and/or name), additional information (identifier and/or value (or value range) and/or designation), external reference, - the loan type (code and/or designation), - the last instalment date (or a date range), - the implementation date (or date range), - the status, - the processing code, - the duration: number (or range) of months, - the amount (or an amount range). For each file returned, all data used in the search criteria will be indicated as well as: - the remaining principal, - the interest rate and APR, - the frequency of instalments, - the number of remaining instalments, - the amount of unpaid amounts.

Service SOAP : createAmortizableLoanFinalValidationStep
Code retour succes : 201 - Created
Uri : http://soprabanking.com/amplitude/loans/v1/{loanId}/finalvalidationsteps
Description : The service createAmortizableLoanFinalValidationStep allows performing the final bank or customer validation step for an instalment loan file corresponding to a unique identifier received. This service is limited to a limited functional scope. It allows adding accounts linked to the loan file, which are necessary to the end of day process, legal attributes of loan file, fund release (only for files with a funds release method at implementation) and implementation charges repayment method. The following features are not processed during the final validation of the instalment loan file by the generic business service: - VISA credits, - Successive validations, - Document check as part of the AIC interface, - Electronic document management, - Validation limits - File validation history, - Exchange rate type and release exchange rate type - File revaluation - APR smoothing of file - Update LSP - Modification of financing source - Check of life or housing insurance (specific ABT) - Collateral check - Modification of pro forma invoices - Printout of acts and printout of schedule - Calculation of file repayment schedule, calculation of file APR and calculation of PARI file, - Modification of negotiated rates, ancillary costs and external charges of file - Modification of file instalment progression - Subsidised loan management - Modification of deferred amount distribution - Check of usury rate - Modification of annual management commission and commitment commission - Management of multiple repayment account - Automatic generation of overdraft limit on commitment account - Event generation CRSOUR - File forbearance - Update of internal insurance characteristics - IAS (fair-value and market rate) - Transact. file validation
