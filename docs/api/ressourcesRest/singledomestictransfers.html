Service SOAP : cancelSingleDomesticTransfer
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/singledomestictransfers/v1/{transferId}
Description : The cancelSingleDomesticTransfer API allows you to cancel a domestic transfer. The API must receive the identifier of the original request in order to recognize and validate the original transfer. It is also necessary to indicate the reason for cancellation of the transfer. The API will return the status of the transfer and indicate whether the transfer has been created or not, and the reason for rejection encountered when the transfer wasn¿t created. Here is the list of the different statuses: ACTC : Transfer correctly integrated but not processed ACSP : Integrated and properly processed transfer PDNG : Transfer processed but with an error (The error encountered will be described in the response) RJCT : Transfer rejected

Service SOAP : createSingleDomesticTransfer
Code retour succes : 201 - Created
Uri : http://soprabanking.com/amplitude/singledomestictransfers/v1/
Description : The API createSingleDomesticTransfer allows you to create a simple domestic transfer between two accounts with an amount and a reason for the transfer. The transfer may be for internal use (same agency or other agency) or external use (other bank). However, this transfer must remain within the local perimeter, that is to say to a local bank. The API will return the status of the transfer and indicate whether the transfer has been created or not, and the reason for rejection encountered when the transfer wasn¿t created. This is the list of the different statuses: ACTC : Transfer correctly integrated but not processed ACSP : Integrated and properly processed transfer PDNG : Transfer processed but with an error (The error encountered will be described in the response) RJCT : Transfer rejected

Service SOAP : createSingleDomesticTransferValidation
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/singledomestictransfers/v1/{transferId}/validations
Description : The createSingleDomesticTransferValidation API allows you to validate a simple domestic transfer created on pending status between two accounts with an amount and a reason for transfer. The transfers must first be created with the "WAIT_CONFIRM" option which allows to create a fund reservation. This option is mainly used for instant payment transfers to block funds if the transfer is accepted. The API must receive the identifier of the original request in order to recognize and validate the original transfer. The API will return the status of the transfer and indicate whether the transfer has been created or not, and the reason for rejection encountered when the transfer wasn¿t created.
