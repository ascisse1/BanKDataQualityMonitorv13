Description : The sub-resource steps enable to process the steps of a loan instruction file.
Service SOAP : createLoanInstructionStepToProcess
Code retour succes : 201 - Created
Uri : http://soprabanking.com/amplitude/loaninstructions/v1/{loanInstructionIdentifier}/steps
Description : The verb POST applied to URI /loaninstructions/v1/{loaninstructionId}/steps allows to take a decision on the current step of a loan instruction file. The decision should be consistent with the parameterization of the current step for the file. The user should have one of the expected roles for processing the mandatory or automatic level. A list of reserves can be sent if the decision concerns the last step level and if the action of agreement under reserve has been activated for the step. The reserves should be relevant for the file type. The rejection reason is to send only if it is activated on the step. Concerning the returned information: - The new step is returned only if a change was done - The new situation is returned only if a change was done - Actions before decision result is returned only if actions before decision were parameterized on the type - Actions after decision result is returned only if actions after decision were parameterized on the type.
