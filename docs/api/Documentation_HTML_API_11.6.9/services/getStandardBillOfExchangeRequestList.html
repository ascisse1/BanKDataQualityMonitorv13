Tables amplitude :

Ce service permet de lister les demandes de carnets de lettres de change normalisées (LCN) correspondant aux critères de sélection contenus dans le flux XML qu'il reçoit.

Afin de limiter le nombre d’enregistrements sélectionnés et d’éviter le déchargement d’une table dans son intégralité, plusieurs combinaisons de critères obligatoires sont définies. Au moins l’une de ces combinaisons doit être donnée en entrée au service pour permettre une restitution de résultats.

Le tableau ci-dessous liste ces combinaisons, où une ligne contient une combinaison possible :

Combinaison de critères

Client (Balise customerNumber au bout du chemin d’éléments suivant : customer.customer)

Numéro de demande (Balise requestNumber)

Compte au format interne (Balises account.internalFormatAccountOurBranch suivantes :

branch.code

et

(currency.alphaCode ou currency.numericCode)

et

account)

Compte au format IBAN (Balise account.ibanFormatAccount.value)

Compte au format externe (Balise account.externalFormatAccount.value)
