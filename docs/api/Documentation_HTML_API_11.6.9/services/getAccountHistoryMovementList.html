Service générique : getAccountHistoryMovementList

Tables amplitude :
  • bkope
  • bkcom

Ce service permet de lister les mouvements de l'historique d'un compte correspondant aux critères de sélection contenus dans le flux XML qu'il reçoit. Pour chaque enregistrement sélectionné, les données caractéristiques d'un mouvement sont renvoyées.

Afin de limiter le nombre d’enregistrements sélectionnés et d’éviter le déchargement d’une table dans son intégralité, plusieurs combinaisons de critères obligatoires sont définies. Au moins l’une de ces combinaisons doit être donnée en entrée au service pour permettre une restitution de résultats.

Le tableau ci-dessous liste ces combinaisons, où une ligne contient une combinaison possible :

Identifiant du compte (Balises branch et currency et account de l’élément accountIdentifier)

Date de début (Balise startDate)

Code module (Balise moduleCode)

Référence du dossier (Balise fileReference)
