Programme Amplitude : cbwprdsubc
Service générique : createProductSubscription

Tables amplitude :
  • bkprdcli
  • bkcom

Ce service permet de souscrire un produit d'attribut divers pour un client correspondant aux critères de sélection contenus dans le flux XML qu'il reçoit. La référence de souscription au produit est ensuite renvoyée. Les relations commerciales du client ne sont pas considérées dans ce service.

 

Concernant le traitement fonctionnel et les règles de gestion, se référer aux documents utilisateurs :

Nom : productCode
Description : Code produit.
Commentaire :

Les vérifications suivantes sont effectuées sur le code produit :

  • Le produit existe et est toujours proposé à la vente.
  • Le produit est configuré pour la vente en produit unitaire.
  • Il s'agit d'un produit divers (attribut du produit à 91) et hors gestion par dossier (gestion par dossier à N).
  • Le produit peut être intégré au package si le code package est renseigné (composant possible d’un package à O).
  • Si le paramètre FATCA existe en nomenclature 098, Le client ne peut pas souscrire un produit (ou un package) si son status FATCA n’est pas renseigné (NouB(bkcli.fatca_status) ou si ce statut est bloquant (lib2 != "O" en nomenclature 072 pour le code statut)
  • Si le paramètre FATCA existe en nomenclature 098, Le client ne peut pas souscrire un produit (ou un package) si son status CRS n’est pas renseigné (NouB(bkcli.crs_status) ou si ce statut est bloquant (lib2 != "O" en nomenclature 072 pour le code statut)
  • Le produit est éligible pour le client : pour les particuliers, le type de clientèle éligible doit être à 1, 5 ou 6 ; pour les sociétés, le type de clientèle éligible doit être à 2, 4 ou 5 ; pour les entreprises individuelles, le type de clientèle éligible doit être à 3, 4, 5 ou 6.

Les valeurs du type de clientèle sont :

  • 1 = Particuliers
  • 2 = Société
  • 3 = Entreprise individuelle
  • 4 = Société et entreprise individuelle
  • 5 = Tous clients
  • 6 = Particulier et entreprise individuelle
    • Le client peut prétendre au produit si :
    • son âge est dans la tranche définie pour le produit.
    • la quantité totale souscrite (par package ou produit unitaire) par le client ne dépasse pas la quantité maximum autorisée pour ce produit

Nom : pricingCode
Description : Code tarification.
Commentaire :

Le code de tarification ne doit pas être renseigné si le produit n'est pas facturable.

Un produit seul est défini comme facturable si le produit est paramétré comme facturable.

Un produit souscrit dans le cadre d'un package est défini comme facturable selon les caractéristiques du produit dans le package.

Si le produit est facturable :

  • Si le code de tarification est renseigné : dans le cas d’une tarification préférentielle hors package, le client doit posséder un package donnant lieu à une tarification préférentielle, sinon dans les autres cas, le code de tarification doit exister et être applicable à la date comptable.
  • Si le code de tarification n'est pas renseigné, alors le tarif à appliquer est sélectionné suivant les cas et par ordre de priorité suivant :
    • Le code package est renseigné et il existe une tarification préférentielle pour le produit.
    • Le code package n’est pas renseigné :
      • Il existe une tarification préférentielle pour le produit si le client possède un certain package, applicable à la date comptable
      • Il existe une tarification de préférence pour le produit, applicable à la date comptable
      • Il existe au moins une tarification pour le produit, applicable à la date comptable.

Dans le cas où aucune tarification n'est trouvée, alors la souscription est rejetée.
