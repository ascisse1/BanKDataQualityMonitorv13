  • Dans tous les autres cas, elle est initailisée nulle

Dans le cas où la date de première révision est modifiable :

  • Si  la date de première révision n’est pas renseignée, que le dossier est avec taux révisable et que le mode de gestion de la date de première échéance pour le type de dossier est D, alors la date de première révison est affectée à la date de première échéance.
  • La date de première révision doit être supérieure ou égale à la date de mise en place du dossier.
  • Si le mode de gestion de la date de première échéance pour le type de dossier est D ou M, alors la date de première révision doit être comprise dans les bornes minimum et maximum paramétrées dans la condition NBPREV.
