Service SOAP : createDirectDebit
Code retour succes : 201 - Created
Uri : http://soprabanking.com/amplitude/directdebit/v1/
Description : The API createDirectDebit permits the generation of a simple or multiple debit order. It will accept orders composed of 1 to n of withdrawals regardless of the expected method of accounting. The creditor's credit will be cumulated or unitary, depending on the instruction communicated via the 'batchBooking' element. The API will return the status of the direct debit order and indicate whether it is created or, if so, the reason for rejection encountered when it was created.Below, find the list of the different statuses: ACTC : Transfer correctly integrated but not processed ACSP : Integrated and properly processed transfer PDNG : Transfer processed but with an error (The error encountered will be described in the response) RJCT : Transfer rejected In order to avoid reaching the timeout and having a lack of response on the transmitter side, we have provided two processing options: synchronous and asynchronous. These options depend on the number of transactions processed in the direct debit order: - If the number of direct debit transactions is below the synchronous limit, it is planned to process for each transaction the data control and the generation of events as well as the associated updates (ACSP response possible). - If the number of direct debit transactions is greater than the synchronous limit and less than the asynchronous limit, it is only intended to process for each transaction the data control (ACSP response impossible). - If the number of direct debit transactions is greater than the asynchronous limit, it is only intended to process an automatic rejection (RJCT response).
