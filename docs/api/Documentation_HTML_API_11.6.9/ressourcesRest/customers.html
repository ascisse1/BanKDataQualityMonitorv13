Service SOAP : getCustomerDetail
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/customers/v1/{customerId}
Description : The service getCustomerDetail allows getting detailed information on a customer corresponding to a unique identifier received. For the selected customer, the following information is returned: - details of the customer, - for customers of 'company' type: . group and sub-group information, . profession and sub-profession information, - details of addresses of the customer, - the list of telephone numbers of the customer, - the list of email addresses of the customer, - details of the free attributes of the customer, - the active profile of the customer.

Service SOAP : createCustomer
Code retour succes : 201 - Created
Uri : http://soprabanking.com/amplitude/customers/v1/
Description : The service createCustomer allows creating a customer. The customer code (identifier) to be created is either provided or automatically allocated. Detail information on the customer is required. It can come along with other optional information from the customer data sheet: - the customer memo, - the list of joint accounts, - the list of children, - the customer's budget expenses and/or incomes), - the list of professions and incomes, - the list of leaders, - the list of shareholders, - the list of financial data, - the list of free attributes (general and/or specific and/or reporting and/or marketing and/or FATCA), - the list of addresses, - the list of telephone numbers, - the list of email addresses, - the list of contact persons, - the customer profile, - the customer's assets, - the list of identity records, - the list of documents, - the list of international operations, - information on the customer's default, - the list of KPP numbers. The service returns the customer code (identifier) created.

Service SOAP : getCustomerList
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/customers/v1/
Description : The service getCustomerList allows returning a list of customers matching the search criteria received. The search is based on the optional criteria below: - limited customer information (code and/or name to be displayed), - the customer type, - the customer profile (code and/or designation), - the national identifier of the customer, - the account officer of the customer (code and/or name), - a piece of additional customer information (identifier and/or value (or value range) and/or designation), - the external reference of the customer, - the customer quality (code and/or designation), - the individual customer data (first name and/or last name and/or middle name and/or date of birth (or range of birth dates) and/or identity record (type code and/or type designation and/or number), - the company customer data (corporate name and/or logo and/or trade registration number and/or business licence number and/or tax identity number and/or customer group (code and/or name)). For each customer returned, all data used in the search criteria will be indicated.

Service SOAP : modifyCustomer
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/customers/v1/{customerId}
Description : The service modifyCustomer allows modifying the data of the customer data sheet corresponding to a unique identifier received. The following data can be modified: - customer detail information, - the customer memo, - the list of joint accounts (replaces all the existing joint accounts of the customer), - the list of children (replace all the existing children of the customer), - the customer's budget (the expenses and/or revenues data replace all the existing expenses and/or revenues data of the customer), - the list of professions and incomes (replace all the existing professions and incomes of the customer), - the list of leaders (replace all the existing leaders of the customer), - the list of shareholders (replace all the existing shareholders of the customer), - the list of financial data (replace all the existing financial data of the customer), - the list of free general attributes (replace all the existing free general attributes of the customer), - the list of free specific attributes (replace all the existing free specific attributes of the customer), - the list of free reporting attributes (replace all the existing free reporting attributes of the customer), - the list of free marketing attributes (replace all the existing free marketing attributes of the customer), - the list of free FATCA attributes (replace all the existing free FATCA attributes of the customer), - the list of addresses (replace all existing addresses of the customer), - the list of telephone numbers (replace all the existing telephone numbers of the customer), - the list of email addresses (replace all the existing email addresses of the customer), - the list of contact persons (replace all the existing contact persons of the customer), - the customer profile, - the customer's assets (the asset data replace all the existing asset data of the customer), - the list of identity records (replace all the existing identity records of the customer), - the list of documents (replace all the existing documents of the customer), - the list of international operations (replace all the existing international operations of the customer), - information on the customer's default, - the list of KPP numbers (replace all the existing KPP numbers of the customer). The customer data are stored in history in the impacted tables before processing the update. The modified data are traced in the table of report on customer modifications (storage of the values before and after the update).
