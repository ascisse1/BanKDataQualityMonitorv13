Service SOAP : createAccount
Code retour succes : 201 - Created
Uri : http://soprabanking.com/amplitude/accounts/v1/
Description : The service createAccount allows creating an account linked to a customer, a file or a product. To create a customer account, the account characteristics are required. They can come along with other optional information linked to the account: - the addresses, - the telephone numbers, - the contact persons, - the email addresses, - additional data. For the creation of an account linked to a product with 'ACCOUNT' attribute, the product subscription is saved after checking the customer is eligible to the subscription. The service returns the identifier of the account created.

Service SOAP : getAccountList
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/accounts/v1/
Description : The service getAccountList allows returning a list of accounts matching the search criteria received. The search is based on the optional criteria below: - the account category, - the code and/or the description of the official branch, - the account identifier, - the account class, - the customer information (identifier, name, type, code and/or designation of the active profile, national identifier, code and/or name of the account officer, additional information and/or external reference of the customer), - a piece of additional account information, - the code and/or the designation of the account type, - the product code and/or designation, - the status, - a range of accounting balances, - the account title. For each account returned, all data used in the search criteria will be indicated as well as: - the instant balance, - the opening date, - the closing date or pending closing status date, - the check key, - the account check digit.

Service SOAP : getAccountActivity
Code retour succes : 200 - OK
Uri : http://soprabanking.com/amplitude/accounts/v1/{accountId}/activities
Description : The service getAccountActivity allows returning the list of movements in an account, whether they are movements of the day or history movements. The search is based on the mandatory criteria below: - the maximum number of movements returned in each list, - the account identifier, - a period composed of dates and times according to the movements wanted and on the optional criteria below: - the module code, - the file reference. The service returns: - the list of history movements for the account according to the period, - the list of movements of the day in the account according to the period.
