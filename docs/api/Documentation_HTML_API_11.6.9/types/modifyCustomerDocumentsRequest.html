Nom : modifyCustomerDocumentsRequest
Description : Requête de modification des documents d'un client.
Choix : Non
  • modifyCustomerDocumentsRequest

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : customerCode
Description : Code du client.
Commentaire :

Le code client n’est pas obligatoire si le service de modification des documents d’un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

S’il est renseigné, il doit correspondre à un client existant dans la table des clients.

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : document
Description : Informations d'un document du client.
Limite du nombre d'occurences mini-maxi : aucune aucune
Commentaire :

Le service "modifyCustomerDocuments" procède par "annule et remplace". Il est donc nécessaire de passer la totalité des documents du client.

Le nombre maximum de documents autorisés pour un client est de 200.

Nom : dateOnWhichSupportingDocumentWasProvided
Description : Date de remise du document justificatif.
Commentaire :

La date de remise du document justificatif est saisissable si le Montant 6 est renseigné et différent de 1 dans la nomenclature de numéro égal au type de document.

Si la date de remise du document justificatif n’est pas renseignée et que le Montant 6 est renseigné à 1 dans la nomenclature de numéro égal au type de document, alors elle est complétée avec la date comptable.
