Nom : accountIdentifier
Description : Compte autre banque avec système de prélèvement télé-compensé.
Commentaire :

Le format interne du compte est prioritaire par rapport au format IBAN, si une balise est renseignée pour le format interne, alors l’IBAN est ignoré.

Le compte d’une autre banque avec système de prélèvement télé-compensé doit être renseigné si :

  • Le paramètre CDTSANSBQE n’existe pas en nomenclature 098 mais les règles de gestion du règlement des impayés par prélèvement pour les dossiers de crédits existent (présence du paramètre CDTRGL en nomenclature 098),
  • Le mode de règlement des échéances est par prélèvement (Libellé 4 renseigné à P en nomenclature 124 pour le mode de règlement de l’emprunteur principal).

Dans ce cas,

  • Le code du client associé au dossier, s’il est renseigné,  doit exister dans la table des clients.
  • Le code de l’établissement, le code du guichet et le numéro de compte doivent être renseignés.
  • Le code de l’établissement ne doit pas correspondre à celui de la banque.
  • Le code de l’agence du dossier, le code de l’établissement et le code du guichet  de ce compte doivent correspondre à une banque existante dans la table des banques.
  • Le numéro de compte doit obligatoirement contenir la clé de contrôle sur les 2 dernières positions. Cette clé RIB est contrôlée si le paramètre CTRL-RIB existe en nomenclature 098 avec le Montant 1 égal à 1, en fonction du code algorithme paramétré dans le Montant 2.

A l'intérieur de ce type, la saisie d'un code BIC implique la saisie d'un code IBAN et inversement.

Ces informations ne doivent être renseignées que si le paramètre CDTSANSBQE est positionné en nomenclature 098.
