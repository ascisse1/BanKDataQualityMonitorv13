Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : branchCode
Description : Code de l'agence.
Commentaire :

Si le code de l'agence est renseigné, alors il doit exister en nomenclature 001.
Sinon, le code du site central est utilisé par défaut.
 

Nom : branchCode
Description : Code d'une agence.
Format : string
Longueur fixe : 5
Nom : tableNumber
Description : Numéro de la nomenclature.
Commentaire :

Le numéro de nomenclature doit être différent de 051.
Le code agence et le numéro de nomenclature doivent correspondre à un enregistrement de la table des identifiants de nomenclatures.
 

Nom : char3
Description : Type avec un nombre de caractères égal à 3.
Format : string
Longueur fixe : 3
Nom : accessKey
Description : Clé d'accès.
Commentaire :

La longueur de la clé d'accès doit respecter le paramétrage donné dans la table des identifiants de nomenclatures.
Contrôle d'existence de la clé d'accès ; si celle-ci n'existe pas déjà, un message d'information sera affiché dans le flux de réponse. Si elle existe, un message d'erreur sera affiché dans le flux et le contrôle sera faux.
 

Nom : charMax10
Description : Chaîne composée d'au plus 10 caractères.
Format : string
Longueur mini-maxi : 1 - 10
Nom : wording1
Description : Libellé 1.
Commentaire :

Un contrôle supplémentaire est fait s'il existe un enregistrement correspondant dans la table des contrôles des nomenclatures. Ce contrôle vérifie que le libellé respecte les valeurs définies dans la table des conversions des messages pour le code stipulé.
