Nom : loanTypeCode
Description : Type de prêt.
Commentaire :

Le type de prêt doit exister dans la table des types de prêts.

Si le type de prêt est paramétré avec gestion des intérêts bonifiés (prêt bonifié à O dans la table des types de prêts) alors, une fois le plan d'amortissement calculé, les échéances de prêts bonifiés réglées par le territoire sont automatiquement renseignées, dans la table des intérêts bonifiés, à partir du pourcentage extrait de la condition PABttt ou PABREP (où ttt correspond au code du type de prêt) et du montant d'intérêts total de chaque échéance.

Pour ce type de prêt, le mode de gestion des comptes pour chaque nature (hormis la nature CDT005 compte de remboursement) doit être Création automatique et chaque nature ne doit être associée qu’à un seul chapitre comptable.

Nom : customerCode
Description : Code client.
Commentaire :

Dans le cas d’une modification d’un dossier de crédit amortissable,  le code du client n'est pas modifiable s’il avait déjà été renseigné lors de la création ou d’une dernière modification du dossier. En revanche, il peut être renseigné identique à celui déjà existant sur  le dossier.

La longueur du code client doit respecter le paramétrage mis en place dans le Montant 1 de la nomenclature 999 pour la clé CLIENT.

Si le client est renseigné alors :

  • Contrôle d’existence du code client dans la table des clients
  • Contrôle des habilitations de l'utilisateur par rapport au profil de clientèle
  • Contrôle au niveau des chapitres autorisés
  • Contrôle que le client ne soit pas sous protocole

Si le  Montant 5 du paramètre CDTPAR2 en nomenclature 098 est égal à 1, les 3 contrôles d'habilitations ci-dessus peuvent être ignorés.

Le client ne doit pas avoir d’opposition positionnée, sauf si le forçage est autorisé  (si la valeur par défaut deropp l’autorise (O) et si le forçage est demandé).

Nom : overallCustomerAuthorizationCode
Description : Numéro d'autorisation plafonnée.
Commentaire :

Si l’autorisation plafonnée n’est pas renseignée et que le client n’en possède qu’une seule valide, alors le numéro d’autorisation est valorisé avec celui de l’autorisation plafonnée du client.

L’autorisation plafonnée est obligatoire selon le paramétrage indiqué pour le type de crédit.

Le numéro d’autorisation doit être numérique.

Si le numéro d’autorisation est renseigné, alors :

  • Il doit exister.
  • La date de début doit être inférieure ou égale à la date comptable.
  • L’autorisation ne doit pas être annulée.
  • La situation ne doit pas être en attente.
  • L’autorisation ne doit pas être échue.

Si le paramètre CDTPAR est positionné en nomenclature 098, alors le code produit de l’autorisation est contrôlé et doit respecter le paramétrage en place.

En cas de dépassement du montant de l'autorisation (du prêt en cours de calcul ou de la somme des en-cours de prêts déjà existants sur le client et rattachés à la même autorisation) ou de la date de l'autorisation, si le forçage de l'autorisation est paramétré (Taux 1 du paramètre CDTPAR4 en nomenclature 098) alors :

  • Si le forçage automatique du dépassement de l'autorisation est indiqué à NON alors la création du dossier est bloquée
  • Sinon, le forçage automatique du dépassement de l'autorisation est indiqué à OUI et la création du dossier peut continuer

Si le dépassement d'autorisation a été forcé, alors un évènement, de nature CDTAPL du montant de dépassement de l'autorisation, est généré.

Nom : overallCustomerAuthorizationExcessLimitOverride
Description : Forçage du dépassement de l'autorisation plafonnée.
Commentaire :

S'il y a un dépassement du montant de l’autorisation ou si la date de dernière échéance dépasse la date d’échéance de l’autorisation, alors :

  • Si le Taux 1 du paramètre CDTPAR4 en nomenclature 098 est égal à 1, alors :
    • Si le forçage est désactivé (booléen à FALSE), alors une erreur est déclenchée.
    • Sinon, si le forçage est activé (booléen à TRUE), alors un avertissement est déclenché.
  • Si le Taux 1 du paramètre CDTPAR4 en nomenclature 098 est égal à 0, alors une erreur est déclenchée.
Nom : boolean
Nom : freeReference
Description : Référence libre.
Commentaire :

Si le paramètre CDTREF n’est pas positionné en nomenclature 098, alors la saisie de la référence libre est interdite. Sinon :

  • Si la référence est renseignée et que sa saisie est automatique ou modifiable et que le mode modification de dossier (hors simulation) a été sélectionné, alors une erreur est renvoyée et aucun autre contrôle n’est fait.
  • Si la référence est renseignée automatiquement (Libellé 2 du paramètre CDTREF en nomenclature 098 positionné à A), alors la référence est affectée avec le code agence, le code banque et la clé de contrôle. Sinon la référence est contrôlée à partir du paramétrage indiqué pour le paramètre CDTREF en nomenclature 098 :
    • Le nombre de caractères de la référence ne doit pas dépasser celui spécifié dans le Montant 1.
    • Si le Libellé 2 est positionné à O, alors la référence libre est obligatoire.
    • Si le Libellé 4 est positionné à O, alors la référence doit être unique pour le dossier de prêt (c'est-à-dire que cette même référence ne doit pas déjà exister pour un autre dossier).
    • Si le Libellé 5 est positionné à O, alors la référence n'est pas modifiable (cas d'une modification du dossier hors modification à l'étape de simulation).
    • Si le Libellé 3 est positionné à O, alors le code de l’agence, le code de la banque et la clé de contrôle sont concaténés à la référence renseignée, avec :
      • Le code agence paramétré dans le paramètre CDTREF en nomenclature 823 ou l'agence du dossier.
      • Le code banque paramétré dans le paramètre CDTREF en nomenclature 840 ou le site central.
      • La clé de contrôle est calculée dans cette condition (Libellé 3 positionné à O dans le paramètre CDTREF en nomenclature 098).
