  • modifyCustomerContactsRequest
Nom : modifyCustomerContactsRequest
Description : Requête de modification des contacts d'un client.
Choix : Non
  • modifyCustomerContactsRequest
    • customerCode : customerCode
    • [M] customerContact : customerContactInformations
      • name : charMax36
      • firstName : charMax30
      • phoneNumber : charMax20
      • emailAddress : charMax50

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : customerCode
Description : Code client.
Commentaire :

Le code client n’est pas obligatoire si le service de modification des contacts d’un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

S’il est renseigné, le code client doit exister dans la table des clients.

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : customerContact
Description : Contact du client.
Limite du nombre d'occurences mini-maxi : aucune aucune
Commentaire :

Le nombre maximum de contacts autorisés pour un client est de 99.

Nom : customerContactInformations
Description : Contact du client.
Nom : name
Description : Nom du contact.
Commentaire :

Le nom du contact doit être renseigné pour permettre l’enregistrement de l’information en base de données.

Valeur par défaut : ctnom
