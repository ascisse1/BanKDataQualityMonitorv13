  • paymentStatus
Nom : paymentStatus
Description : Etat d'un règlement.
Format : string
Valeurs possibles :
10 Règlement en attente d'exécution
00 Règlement initié
90 Règlement abandonné
40 Règlement reversé au fournisseur
30 Règlement exécuté validé
20 Règlement exécuté en attente de forçage

Liste des valeurs possibles :

  • 00 = Règlement initié
  • 10 = Règlement en attente d'exécution
  • 20 = Règlement exécuté en attente de forçage
  • 30 = Règlement exécuté validé
  • 40 = Règlement reversé au fournisseur
  • 90 = Règlement abandonné
