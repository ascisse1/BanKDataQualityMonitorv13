Commentaire :

Valeur par défaut : revproech

Nom : boolean
Nom : firstReview
Description : Date de la première révision.
Commentaire :

La date de la première révision n’est pas modifiable si :

  • le mode de gestion de la date de première échéance pour le type de dossier est M ou D et les dates minimum et maximum de première révision par défaut sont nulles ou égales (les dates minimum et maximum de première révision sont calculées en fonction de la date de mise en place du dossier et de la condition NBPREV si le mode de gestion de la date de première échéance pour le type de dossier est M ou D, sont affectées à NULL sinon) – indépendamment du Libellé 3 du paramètre CDTREVIS en nomenclature 098
  • la méthode est Taux à la date de révision (R) et le  Libellé 1 est égal à N du paramètre CDTREVIS en nomenclature 098
  • la méthode est Taux à la date de début de période (P) et le  Libellé 2 est égal à N du paramètre CDTREVIS en nomenclature 098,
  • la méthode est Tous les changements (C) et le  Libellé 3 est égal à N du paramètre CDTREVIS en nomenclature 098

Dans ce cas,

  • si la méthode de révision est Taux à la date de début de période (P) et que le Libellé 4 du paramètre CDTREVIS en nomenclature 098 est égal à O, la date de première révision est affectée à la date de mise en place du dossier (ou premier jour ouvré suivant),
  • sinon, si la date de première échéance du dossier n'est pas nulle et que la méthode de gestion du différé est ni   Echéance (E), ni Spécial (S) et ni Echéances intercalaires avec date limite d’utilisation (I), alors la date de première révision est affectée à la date de première échéance.
  • Dans tous les autres cas, elle est initailisée nulle

Dans le cas où la date de première révision est modifiable :

  • Si  la date de première révision n’est pas renseignée, que le dossier est avec taux révisable et que le mode de gestion de la date de première échéance pour le type de dossier est D, alors la date de première révison est affectée à la date de première échéance.
  • La date de première révision doit être supérieure ou égale à la date de mise en place du dossier.
  • Si le mode de gestion de la date de première échéance pour le type de dossier est D ou M, alors la date de première révision doit être comprise dans les bornes minimum et maximum paramétrées dans la condition NBPREV.

Nom : lastReview
Description : Date de la dernière révision.
Commentaire :

La date de la dernière révision n’est pas modifiable si :

  • la méthode est Taux à la date de révision (R) et le  Libellé 1 est égal à N du paramètre CDTREVIS en nomenclature 098
  • la méthode est Taux à la date de début de période (P) et le  Libellé 2 est égal à N du paramètre CDTREVIS en nomenclature 098,
  • la méthode est Tous les changements (C) et le  Libellé 3 est égal à N du paramètre CDTREVIS en nomenclature 098

Dans ce cas, elle est affectée avec la date de dernière échéance si elle existe.

Dans le cas où la date de dernière révision est modifable, elle doit être :

  • renseignée si la date de première révision est renseignée,
  • supérieure ou égale à la date de première révision,
  • inférieure ou égale à la date de dernière échéance du dossier.

Nom : reviewCode
Description : Code de révision.
Commentaire :

Le fait de renseigner un code de révision implique que le tableau de progression des intérêts va être calculé automatiquement.

Les données du tableau de progression des intérêts ne doivent donc pas être renseignées dans ce cas.

Si ni le code de révision, ni le tableau de progression des intérêts sont renseignés, le code de révision sera affecté par défaut avec un code de révision du type de prêt (un code de révision dont le(s) code(s) de taux de base lié(s) au code de révision sont cohérents avec la devise du dossier).

Le code de révision renseigné doit exister :

  • dans la table des codes pour les taux d’intérêts révisables associés au type de prêt (soit au niveau de l'agence du dossier soit au niveau du site central),
  • dans la table des entêtes des codes de taux pour les intérêts révisables.

Le détail du code révision doit être paramétré dans la table du détail des lignes des taux. Un code taux, taux de base, doit exister dans au moins un enregistrement de cette table parmi tous les enregistrements existants pour le code révision.

Ce code de taux de base doit également exister pour la devise du prêt et le type de prêt, dans la table des codes de taux de base liés à un type et une devise.

Nom : date
Nom : baseRate
Description : Code du taux de base.
Commentaire :

Pour la révision, soit le code du taux de base, soit un taux fixe doit être renseigné.

Pour la première révision de taux renseignée (celle dont la date d'application est égale à la date de mise en place du dossier), si ni le code du taux de base n’est renseigné ni le taux fixe, alors le code du taux de base sera affecté par défaut avec le code paramétré pour la condition TBPRET.

Si un code du taux de base est modifié mais que le niveau de forçage de l'utilisateur ne lui permet pas de le modifier par rapport au niveau de forçage paramétré pour le type de prêt, alors le code du taux de base sera affecté avec celui par défaut.

Si le code du taux de base n’est pas géré par condition (Libellé 2 égal à N dans le paramètre CDTPAR2 en nomenclature 098) alors le code du taux de base doit être autorisé parmi une liste de codes possibles en fonction de la devise et du type de crédit (dans la table des entêtes des taux de base et dans la table des codes des taux liés à un type de prêt et une devise).

Sinon, le code du taux de base doit exister uniquement dans la table des entêtes des taux de base.

Si la gestion des tranches de maturité est activée (paramétrage des tranches en nomenclature 156), alors le code maturité du taux de base doit être égal au code maturité du dossier.

Si le code du taux de base est renseigné :

  • Si le Montant 4 du paramètre CDTREVIS en nomenclature 098 est égal à 1, alors la marge ne doit pas être renseignée mais uniquement le taux final (la valeur de la marge est alors calculée par différence entre le taux final et la valeur du taux de base)
  • Si le Montant 4 du paramètre CDTREVIS en nomenclature 098 vaut 0 ou NULL, alors le taux final ne doit pas être renseigné mais uniquement la marge (a valeur du taux final est alors calculée par addition entre la valeur du taux de base et la marge)

Si le code du taux de base n'est pas renseigné, alors ni la marge ni le taux final ne doivent être renseignés. Seul le taux fixe doit être alimenté.

Nom : margin
Description : Marge.
Commentaire :

La marge ne doit être renseignée que si le taux de base est renseigné pour la révison et que la marge est autorisée (Montant 4 du paramètre CDTREVIS en nomenclature 098 différent de 1).

Si la marge peut être renseignée mais qu’elle ne l’est pas, alors elle est affectée à la valeur  de la condition VTBPRG.

La marge doit être comprise dans les bornes minimum et maximum paramétrées dans la condition VTBPRG.

Le taux final est alors calculé automatiquement. Et il doit être :

  • Supérieur ou égal au taux de révision minimum,
  • Inférieur ou égal au taux de révision maximum.
