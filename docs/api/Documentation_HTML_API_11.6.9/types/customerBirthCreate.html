Nom : birthDate
Description : Date de naissance.
Commentaire :

Si le contrôle est demandé avec l’existence du paramètre CONTROLDNA en nomenclature 098, l’âge du client ne doit pas excéder l’âge maximum paramétré dans le Montant 1.

La date de naissance ne doit pas être supérieure à la date comptable.

Si le contrôle d’unicité est demandé dans le Libellé 4 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 3)) pour le paramètre CTRIDENCL2 mais pas dans le Libellé 2 (différent de O et B) pour le paramètre CTRIDENCL3 de la nomenclature 098, alors il ne doit pas exister de client possédant les mêmes nom, prénom, deuxième prénom et date de naissance.

Si le contrôle d’unicité du nom pour la Banque de France est demandé dans le Libellé 5 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 4)) pour le paramètre CTRIDENCLI de la nomenclature 098, il ne doit pas exister de client avec les mêmes nom (ou nom de jeune fille si le sexe du client est féminin (F)) et date de naissance.

Nom : date
Nom : birthCity
Description : Ville de naissance.
Commentaire :

Si la ville de naissance ou le nom de la mère sont renseignés, et si le contrôle de vérification des doublons est demandé dans le Libellé 2 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 1)) pour le paramètre CTRIDENCL3 de la nomenclature 098, il ne doit pas exister de clients possédant les mêmes nom, prénom, date de naissance, ville de naissance et/ou nom de mère.

Sinon, si le contrôle de vérification des doublons est demandé dans le Libellé 4 (égal à O (contrôle non bloquant) ou B (contrôle bloquant uniquement si l’utilisateur n’a pas le niveau de forçage requis indiqué dans le Montant 3)) pour le paramètre CTRIDENCLI de la nomenclature 098, il ne doit pas exister de clients possédant les mêmes nom, prénom, date de naissance et ville de naissance.

Si les contrôles d’existence et de gestion en base de la ville sont demandés respectivement dans le Montant 3 et le Libellé 2 du paramètre VILLE en nomenclature 098, alors la ville doit exister dans la table des villes pour le département de naissance et la région de naissance s’ils sont renseignés.

Si le pays n’est pas un pays étranger (Montant 3 renseigné ou Montant 2 de la nomenclature 040 différent de 4) alors on recherche le code localité de la ville de naissance dans la table des villes.

Valeur par défaut (contrôle de l’obligation de saisie uniquement) : viln
