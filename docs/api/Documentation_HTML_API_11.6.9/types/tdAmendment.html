Nom : amendmentAccount
Description : Compte de l¿avenant.
Commentaire :

Le compte doit être dans la devise du dépôt ou dans la devise nationale.
Le compte ne doit pas être un compte de dépôt à terme.
Le compte doit appartenir au même client que le dépôt à terme.

Nom : valueDate
Description : Date de valeur de l¿avenant.
Commentaire :

La date de valeur doit être :

  • ¿ Supérieure à la date de valeur du dépôt à terme.
  • ¿ Inférieure à la date d’échéance du dépôt à terme
  • ¿ Supérieure à la dernière date de versement d’intérêts
  • ¿ Supérieure à la dernière date de calcul d’avenant
  • ¿ Comprise entre les dates minimum et maximum si la condition NBJDAV existe et si sa valeur est différente de 0.

Il ne doit pas exister d’avenant à cette date.

Nom : amendmentSide
Description : Sens de l'avenant.
Commentaire :

Si le dépôt à terme est nanti ou si le dépôt à terme est à taux progressif, alors le sens de l’avenant doit être au crédit.

Nom : amendmentAmount
Description : Montant de l¿avenant.
Commentaire :

Si le dépôt à terme est à taux progressif, le montant doit être supérieur au montant minimum d’augmentation (valeur de la condition MINAVE).
