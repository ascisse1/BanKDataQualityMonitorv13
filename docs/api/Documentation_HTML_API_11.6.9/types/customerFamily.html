Nom : customerFamily
Description : Famille.

Pour un client de type Particulier (1).

Choix : Non
  • customerFamily
    • spouseType : customerThirdPartyType
    • spouseCode : charMax15
    • numberOfChildren : short
    • customerFamilyRelationshipCode : charMax15

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : spouseType
Description : Type de conjoint.
Commentaire :

Le type de conjoint ne peut être renseigné que si le code du conjoint est renseigné.

Nom : customerThirdPartyType
Description : Population ciblée.
Format : string
Valeurs possibles :
Tous
C Clients
T Tiers
Nom : spouseCode
Description : Code du conjoint.
Commentaire :

Le code du conjoint doit exister dans la table des clients ou dans la table des tiers et doit être cohérent avec le type de conjoint renseigné.

Valeur par défaut (contrôle de l’obligation de saisie uniquement) : conj

Nom : charMax15
Description : Chaîne composée d'au plus 15 caractères.
Format : string
Longueur mini-maxi : 1 - 15
Nom : numberOfChildren
Description : Nombre d'enfants.
Commentaire :

En création ou en modification d’un client, si des enfants sont renseignés dans la balise childrenList, alors :

  • Si le nombre d’enfants renseigné dans la liste est différent du nombre indiqué dans la balise numberOfChildren, alors :
    • Le nombre d’enfants indiqué ici est ignoré et remplacé par le nombre d’enfants effectivement renseignés dans la liste de la balise childrenList.

En modification d’un client, si aucun enfant n’est renseigné dans la balise childrenList et que le nombre d’enfants est différent de celui déjà associé au client, alors il sera ignoré et remplacé par ce dernier.

Valeur par défaut : nbenf

Nom : short
Nom : customerFamilyRelationshipCode
Description : Code du client de rattachement familial.
Commentaire :

Si le code client de rattachement familial est renseigné, alors il doit exister dans la table des clients.

Valeur par défaut (contrôle de l’obligation de saisie uniquement) : clifam

Nom : charMax15
Description : Chaîne composée d'au plus 15 caractères.
Format : string
Longueur mini-maxi : 1 - 15
