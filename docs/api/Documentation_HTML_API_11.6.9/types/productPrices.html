Nom : pricingCode
Description : Code tarification.
Commentaire :

Le code de tarification ne doit pas être renseigné si le produit n'est pas facturable.

Un produit seul est défini comme facturable si le produit est paramétré comme facturable.

Un produit souscrit dans le cadre d'un package est défini comme facturable selon les caractéristiques du produit dans le package.

Si le produit est facturable :

  • Si le code de tarification est renseigné : dans le cas d’une tarification préférentielle hors package, le client doit posséder un package donnant lieu à une tarification préférentielle, sinon dans les autres cas, le code de tarification doit exister et être applicable à la date comptable.
  • Si le code de tarification n'est pas renseigné, alors le tarif à appliquer est sélectionné suivant les cas et par ordre de priorité suivant :
    • Le code package est renseigné et il existe une tarification préférentielle pour le produit.
    • Le code package n’est pas renseigné :
      • Il existe une tarification préférentielle pour le produit si le client possède un certain package, applicable à la date comptable
      • Il existe une tarification de préférence pour le produit, applicable à la date comptable
      • Il existe au moins une tarification pour le produit, applicable à la date comptable.

Dans le cas où aucune tarification n'est trouvée, alors la souscription est rejetée.

Nom : billingAccount
Description : Compte de facturation.
Commentaire :

Le compte de facturation ne peut être renseigné que si le produit est facturable.

Si non renseigné, le compte est initialisé avec un compte du client correspondant au paramétrage de la nature de compte du code de tarification.

Si renseigné, le compte doit exister, être ouvert et correspondre au paramétrage de la nature de compte du code de tarification.

La devise du compte de facturation doit être en adéquation avec le type de contrôle de la devise de facturation du produit (si le type est à 2, la devise du compte de facturation doit être en devise nationale ; si le type est à 3, la devise du compte de facturation doit être en devise étrangère).

Nom : quantityOfSubscribedProducts
Description : Quantité de produits souscrits.
Commentaire :

La quantité de produits souscrits ne peut être renseignée que si le produit est facturable et que la tarification est paramétrée selon le type quantitatif et manuel.

La quantité doit respecter le paramétrage établi par les conditions PRQMIN et PRQMAX.

Une valeur par défaut est utilisée si la quantité de produits n'est pas renseignée et que la condition PRQDEF est renseignée.

Nom : integer
Nom : billingCharges
Description : Facturation des frais de mise en place.
Commentaire :

L'option "facturation des frais de mise en place" ne peut être activée que si le produit est facturable.

Si le produit est facturable et qu'aucune valeur n'est renseignée pour l'option de facturation des frais, alors les frais de mise en place sont calculés en fonction des conditions paramétrées.

Nom : boolean
Nom : dutyAmountToTheEstablishment
Description : Montant hors taxe à la mise en place.
Commentaire :

Le montant ne peut être renseigné que si le produit est facturable et qu'il permet la facturation de frais de mise en place.
