Nom : dosprtCreateNegociatedRate
Description : Taux d'intérêts négociés.

Il doit exister à minima une condition INNttt, VINttt, INNPRT ou VINPRT pour permettre la saisie d’un taux d’intérêt négocié, ttt correspondant au code du type de prêt.

Choix : Non
  • dosprtCreateNegociatedRate
    • effectiveDate : date
    • rateValue : decimal9_6
    • marginValue : decimal9_6
    • internalRate : decimal9_6
    • nextNegociationDate : date

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : effectiveDate
Description : Date d¿effet du taux.
Commentaire :

La date d’effet du taux doit être égale à la date de mise en place du dossier.

Nom : date
Nom : rateValue
Description : Valeur du taux.
Commentaire :

Si la valeur du taux n’est pas  renseignée, elle est affectée avec la valeur de la condition INNttt où ttt correspond au code du type de prêt ou INNPRT.

La valeur du taux doit être comprise dans les bornes minimum et maximum paramétrées dans la condition INNttt où ttt correspond au code du type de prêt ou INNPRT.

La somme des valeurs du taux et de la marge doit être différente de 0.

Nombre total de chiffres : 9
