Nom : char5

Format : string
Longueur fixe : 5
Nom : organization
Description : Code organisme.
Commentaire :

Le code de l'organisme doit exister en nomenclature 054.
Si le paramètre DIRECT-DEB existe en nomenclature 098, alors si le Montant 1 est renseigné et différent de zéro, alors la clé construite à partir du code organisme précédé de la mention "ORG" doit exister en nomenclature 098.
 

Nom : char5
Description : Type avec un nombre de caractères égal à 5.
Format : string
Longueur fixe : 5
Nom : accountNumber
Description : Numéro de compte.
Commentaire :

Le numéro de compte doit exister dans la table des comptes et respecter le format défini en nomenclature 999.
Le compte ne doit pas être fermé ni en instance de fermeture.
 

Nom : charMax11
Description : Chaîne composée d'au plus 11 caractères.
Format : string
Longueur mini-maxi : 1 - 11
Nom : accountSuffix
Description : Suffixe du compte.
Commentaire :

Le suffixe du compte est testé en même temps que le numéro de compte.
De plus, il doit être numérique.
 

Nom : char2
Description : Type avec un nombre de caractères égal à 2.
Format : string
Longueur fixe : 2
Nom : subscriberNumber
Description : Numéro d'abonné.
Commentaire :

Le numéro d’abonné doit respecter la longueur définie dans le Montant 6 de la nomenclature 054 pour l’organisme donné.
Le numéro d'abonné est obligatoire si la valeur paramétrée pour le Taux 3 de la nomenclature 054 pour l'organisme donné vaut 1 ou 2, et ne doit pas être renseignée si elle vaut 0.
 

Nom : charMax20
Description : Chaîne composée d'au plus 20 caractères.
Format : string
Longueur mini-maxi : 1 - 20
Nom : complementaryIdentifier
Description : Identifiant complémentaire de l'abonné.
Commentaire :

L'identifiant complémentaire doit respecter la longueur définie dans le Montant 8 de la nomenclature 054 pour l'organisme donné.
L'identifiant complémentaire est obligatoire si la valeur paramétrée pour le Taux 3 de la nomenclature 054 pour l’organisme donné vaut 2, et ne doit pas être renseignée si elle vaut 0.
 

Nom : charMax20
Description : Chaîne composée d'au plus 20 caractères.
Format : string
Longueur mini-maxi : 1 - 20
Nom : limit
Description : Plafond de l'autorisation de prélèvement.
Commentaire :

Le plafond doit être strictement supérieur à zéro.

Nom : decimal19_4

Nom : managementFees
Description : Frais de gestion de l'autorisation de prélèvement.

Nom : boolean
Nom : terminationDate
Description : Date de résiliation de l'autorisation de prélèvement.
Commentaire :

La date de résiliation ne doit pas être postérieure à la date comptable du jour.
Aucune autorisation de prélèvement ne doit être en cours à cette date.
