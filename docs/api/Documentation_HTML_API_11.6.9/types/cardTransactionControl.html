Nom : cardTransactionControl
Description : Définition du niveau de contrôle sur une transaction monétique.
Choix : Non
  • cardTransactionControl
    • checkDisapprovals : boolean
    • consistencyWithFundsReservation : cardTransactionConsistencyWithFundsReservation
      • consistencyWithFundsReservationAmount : boolean
      • comparisonAmountVarianceRate : decimal15_7
      • [C] behaviourIfNonConsistencyWithFundsReservation : cardTransactionAccountBehaviour
        • useRejectAccount : boolean
        • useSuspenseAccount : boolean
        • useUnpaidAccount : boolean
        • rejectTransaction : boolean
    • fundsReservationsCancellationRules : cardFundsReservationCancellationRules
      • fundsReservationCancellationIfNonBlockingDisapprovals : boolean
      • eventStatusIfNotCancelledFundsReservation : eventStatus

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : checkDisapprovals
Description : Contrôle des désaccords pour la transaction et ses comptes ?
Commentaire :

Par défaut à FALSE : les désaccords ne sont pas contrôlés.

Nom : boolean
Nom : consistencyWithFundsReservation
Description : Cohérence avec la réservation de fonds
Commentaire :

Par défaut, aucun contrôle de cohérence avec la réservation de fonds n’est réalisé.

Cette balise n'est considérée que si la clé de rapprochement avec la réservation de fonds est fournie dans le flux.

Nom : cardTransactionConsistencyWithFundsReservation
Description : Cohérence avec l'AVIDAB.
Nom : consistencyWithFundsReservationAmount
Description : Contrôle de cohérence du montant de la transaction avec le montant de l'AVIDAB.
Commentaire :

Par défaut FALSE : pas de contrôle de cohérence avec le montant de l’AVIDAB.

Nom : boolean
Nom : comparisonAmountVarianceRate
Description : Taux de variation autorisée pour le contrôle.
Commentaire :

Utilisé uniquement si le contrôle de cohérence des montants est demandé.

Plus le taux est bas, plus les montants doivent être identiques.

Si non renseigné, un taux de variation autorisée par défaut de 0% est utilisé (correspondant à un contrôle d’égalité stricte).

Nom : decimal15_7
Description : Type décimal à 15 chiffres dont 7 décimales.
Format : decimal
Nombre total de chiffres : 15
Nom : behaviourIfNonConsistencyWithFundsReservation
Description : Comportement en cas d'incohérence avec l'AVIDAB.
Commentaire :

Par défaut, la transaction est rejetée.

Nom : cardTransactionAccountBehaviour
Description : Comportement attendu en cas de défaut du compte.

Nom : fundsReservationCancellationIfNonBlockingDisapprovals
Description : Annulation de l'AVIDAB si désaccords non bloquants ?
Commentaire :

Par défaut TRUE : annulation de l'AVIDAB si des désaccords non bloquants sont rencontrés lors du traitement de l’événement de la transaction.

Nom : boolean
Nom : eventStatusIfNotCancelledFundsReservation
Description : Etat de l'événement de transaction si l'AVIDAB n¿a pas été annulé.
Commentaire :

Par défaut IG : ignoré.
