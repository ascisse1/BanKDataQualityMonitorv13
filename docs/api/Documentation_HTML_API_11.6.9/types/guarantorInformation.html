Nom : guarantorInformation
Description : Informations sur un garant.
Choix : Non
  • guarantorInformation
    • category : guarantorCategory
    • alreadyGuarantor : boolean
    • guarantorTitle : titleCode
      • code : charMax2
      • designation : charMax30
    • guarantorCode : charMax15
    • lastName : charMax36
    • firstName : charMax30
    • birthDate : comparisonDate
      • date1 : date
      • date2 : date
      • comparator : comparator
    • [C] additionalDataValue : additionalDataValue
      • alphanumeric : charMax40
      • amountOrRate : decimal22_7
      • date : date

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

La 2ème date n’est prise en compte que pour les opérateurs de comparaison BETWEEN et BETWEEN EQUALS.

Nom : date
Nom : comparator
Description : Opérateur de comparaison.
Commentaire :

Vaut EQUALS dans un flux de réponse.

Nom : comparator
Description : Opérateur de comparaison.
Format : string
Valeurs possibles :
GREATER THAN Supérieur strict
NOT EQUALS Différent de
GREATER EQUALS Supérieur ou égal à
LOWER EQUALS Inférieur ou égal
BETWEEN Compris entre bornes exclues
BETWEEN EQUALS Compris entre bornes incluses
EQUALS Egal à
LOWER THAN Inférieur strict
Nom : additionalDataValue
Description : Valeur de la donnée complémentaire.
Commentaire :

Si le paramétrage GARANTIE existe en nomenclature 098, alors l’identifiant paramétré dans le Libellé 3 est considéré pour la recherche et la restitution de la valeur de la donnée complémentaire.

Sinon, la valeur reçue est ignorée et n’apparaît pas en restitution.

Description : Valeur d'une donnée complémentaire.
