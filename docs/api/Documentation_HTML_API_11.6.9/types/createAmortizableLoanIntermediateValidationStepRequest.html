Nom : amortizableLoanIdentifier
Description : Identifiant d¿un crédit amortissable.
Commentaire :

L'identifiant du dossier de crédit amortissable doit exister dans la table des dossiers de crédit amortissable.

Si la gestion de l’historisation du type de crédit amortissable est activée, alors une étape de validation est insérée dans la table des historiques des crédits.

Si l’étape de validation banque est à comptabiliser, alors :

  • Un compte d'engagement autorisé (de nature CDT002) est créé dans la table des comptes d’un crédit.
  • Un événement DECBQE est ajouté à la table des événements.

Si l’étape de validation client est à comptabiliser, alors :

  • Un compte d'engagement disponible (de nature CDT003) est créé dans la table des comptes d’un crédit, s’il n’a pas déjà été créé lors de la création du dossier de crédit (compte créé si le code autorisation associé au type de dossier de crédit est "gestion d'une autorisation au niveau du compte de nature CDT003 (O)" ou "gestion d'une autorisation au niveau des comptes de natures CDT003 et CDT004 (A)".
  • Un événement DECCLI est ajouté à la table des événements.

Dans le cas de la banque ATTIJARI (existence du paramètre ABT en nomenclature 098), alors les assurances vie ou habitation sont obligatoires.

Le nombre de cautions / co-emprunteurs renseignés sur le dossier doit obligatoirement correspondre au nombre de cautions / co-emprunteurs enregistrés dans la table des intervenants sur les dossiers.

Le dossier de crédit ne doit pas posséder de désaccord bloquant sur le compte de remboursement.

Nom : validationType
Description : Type de validation.
Commentaire :

Le changement de type de validation n’est autorisé que si le dossier est à l’état Dossier en cours d’étude (SI) ou à l’état Dossier validé par la banque (VB).

Si le dossier est en cours d’étude (SI), alors :

  • Le type de validation ne peut être que Dossier validé par la banque (VB) si la validation banque est demandée pour le type de dossier de crédit.
  • Le changement du type de validation ne sera autorisé que si le dossier est complet (code traitement à 1).

Si le dossier est à l’état Dossier validé par la Banque (VB), alors :

  • Le type de validation ne peut être que Dossier validé par le client (VC) si la validation client est demandée pour le type de dossier de crédit.
  • Le changement de type de validation ne sera autorisé que si le dossier est accepté par la banque (code traitement à 1).

Si le type de validation demandé est validation banque (VB) et que la banque possède une interface AIC (présence du paramètre INTERFAIC en nomenclature 098) alors les documents issus de l’interface AIC sont obligatoires si la condition CDTAIC demande ce contrôle.

Si les documents sont gérés au niveau du type de prêt, alors tous les documents obligatoires doivent être fournis

Nom : commissionRate
Description : Taux de la commission d'engagement.
Commentaire :

Par défaut, le taux de la commission d’engagement est affectée avec le taux de  la condition MOEPRT pour le code opération enregistré au niveau du type de prêt.

Les bornes minimale et maximale de la condition doivent être respectées.

Si la commission est taxable et que le client de remboursement est  taxable, alors le taux de cette taxe est affecté avec le taux de  la condition TTAXEN.
