  • chequeAndBillDetailPTE
Nom : chequeAndBillDetailPTE
Description : Détail des effets d'un dossier de Remdoc export.
Choix : Non
  • chequeAndBillDetailPTE
    • chequeOrBillNumber : charMax15
    • amount : decimal19_4
    • termOfPayment : char3
    • maturityDate : date
    • guaranteed : boolean

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : chequeOrBillNumber
Description : Numéro de cheque ou d'effet.
Commentaire :

Si plusieurs effets le numéro d’effet doit être unique.

Nom : charMax15
Description : Chaîne composée d'au plus 15 caractères.
Format : string
Longueur mini-maxi : 1 - 15
Nom : amount
Description : Montant de l'effet.
Commentaire :

La somme totale des effets doit être égale au montant du dossier.

Nom : decimal19_4
Description : Type décimal à 19 chiffres dont 4 décimales.
Format : decimal
Nombre total de chiffres : 19
Nom : termOfPayment
Description : Modalité de paiement.
Commentaire :

Nomenclature 504 avec mnt3=0 ou mnt3=2.

Nom : char3
Description : Type avec un nombre de caractères égal à 3.
Format : string
Longueur fixe : 3
Nom : maturityDate
Description : Date d'échéance.
Commentaire :

Si renseigné, la date d’échéance doit être supérieur ou égale à la date comptable.

Nom : date
Nom : guaranteed
Description : Avalisé.
Commentaire :

Si non renseigné prendre la valeur par défaut pour la zone ‘aval’ définie sur le type de remdoc. Si non renseigné alimenter par défaut à ‘N’.

Nom : boolean
