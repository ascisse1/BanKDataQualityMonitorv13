  • createChequeBookRequestResponse
Nom : createChequeBookRequestResponse
Description : Réponse de la création d'une commande de chéquier.
Choix : Non
  • createChequeBookRequestResponse
    • requestNumber : eventNumber
    • requestDate : date
    • valueDate : date
    • requestSituation : chequeBookRequestSituation
    • requestStatus : chequeBookRequestStatus
    • charges : decimal19_4
    • [M] disapprovals : disapproval
      • code : disapprovalCode
      • designation : charMax30

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : requestNumber
Description : Numéro de commande de chéquier.
Commentaire :

Le numéro de commande de chéquier est valorisé uniquement si la commande a abouti.

Nom : eventNumber
Description : Numéro d'événement.
Format : string
Longueur fixe : 6
Nom : requestDate
Description : Date de commande de chéquier.
Commentaire :

La date de la commande de chéquier est valorisée uniquement si la commande a abouti.

Nom : date
Nom : valueDate
Description : Date de valeur de la commande de chéquier.
Commentaire :

La date de valeur de la commande de chéquier est égale à la date de commande du chéquier.

Nom : date
Nom : requestSituation
Description : Situation de la commande de chéquier.
Nom : chequeBookRequestSituation
Description : Situation de la demande de chéquier.
Format : string
Valeurs possibles :
X Erreur
P personnalisée
A arrivé à l'agence de délivrance
R chéquier remis au client
C commandée
S chéquier détruit
D demandée
E envoyé à l'agence de délivrance
Nom : requestStatus
Description : Etat de la commande de chéquier.
Nom : chequeBookRequestStatus
Description : Etat de la demande de chéquier.
Format : string
Valeurs possibles :
AT en attente
VF validée forcée
AB abandonnée
FO forcée
IF ignorée forcée
IG rejetée
VA validée
Nom : charges
Description : Montant des frais de la commande de chéquier.
Commentaire :

Ce montant est nul s'il n'y a pas de frais.

Nom : decimal19_4
Description : Type décimal à 19 chiffres dont 4 décimales.
Format : decimal
Nombre total de chiffres : 19
