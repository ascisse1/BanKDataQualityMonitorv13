Nom : amortizableLoanIdentifier
Description : Identifiant d¿un crédit amortissable.
Commentaire :

L'identifiant du dossier de crédit amortissable doit exister dans la table des dossiers de crédit amortissable.

Seuls les dossiers incomplets en cours d’étude (code état SI et code contrôle 0), les dossiers complets en cours d’étude (code état SI et code contrôle 1), les dossiers acceptés par la banque (code état VB et code contrôle 1) et les dossiers acceptés par le client (code état VC et code contrôle 1) peuvent être annulés.

Si la gestion de l’historisation du type de crédit amortissable est activée, alors une ligne d’annulation est insérée dans la table des historiques des crédits.

Si l’étape d’étude, l’étape de validation banque ou l’étape de validation client est à comptabiliser, alors un événement ANSOUR est ajouté à la table des événements.

Si le dossier annulé était déjà validé par la banque alors un événement FANPRT est ajouté à la table des événements avec les montants des frais et taxes  paramétrés dans les conditions FANPRT et FANTAX.
