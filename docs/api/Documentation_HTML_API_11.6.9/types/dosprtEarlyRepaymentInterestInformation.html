Nom : dosprtFloatingOrProgressiveRateChange
Description : Détail d'un changement de taux progressif ou révisable.
Nom : applicationDate
Description : Date d'application du taux.
Commentaire :

La première date d’application du taux  (la date la plus ancienne parmi toutes les lignes de révision à traiter) doit être égale à la date de mise en place du dossier.

La date d’application du taux doit être supérieure strictement à la date d’application du taux précédent (à partir de la  2ème ligne du tableau).

La date d’application du taux doit être supérieure ou égale à la date comptable.

La date d’application du taux doit être inférieure ou égale à la date de dernière révision.

Nom : date
Nom : baseRate
Description : Code du taux de base.
Commentaire :

Pour la révision, soit le code du taux de base, soit un taux fixe doit être renseigné.

Pour la première révision de taux renseignée (celle dont la date d'application est égale à la date de mise en place du dossier), si ni le code du taux de base n’est renseigné ni le taux fixe, alors le code du taux de base sera affecté par défaut avec le code paramétré pour la condition TBPRET.

Si un code du taux de base est modifié mais que le niveau de forçage de l'utilisateur ne lui permet pas de le modifier par rapport au niveau de forçage paramétré pour le type de prêt, alors le code du taux de base sera affecté avec celui par défaut.

Si le code du taux de base n’est pas géré par condition (Libellé 2 égal à N dans le paramètre CDTPAR2 en nomenclature 098) alors le code du taux de base doit être autorisé parmi une liste de codes possibles en fonction de la devise et du type de crédit (dans la table des entêtes des taux de base et dans la table des codes des taux liés à un type de prêt et une devise).

Sinon, le code du taux de base doit exister uniquement dans la table des entêtes des taux de base.

Si la gestion des tranches de maturité est activée (paramétrage des tranches en nomenclature 156), alors le code maturité du taux de base doit être égal au code maturité du dossier.

Si le code du taux de base est renseigné :

  • Si le Montant 4 du paramètre CDTREVIS en nomenclature 098 est égal à 1, alors la marge ne doit pas être renseignée mais uniquement le taux final (la valeur de la marge est alors calculée par différence entre le taux final et la valeur du taux de base)
  • Si le Montant 4 du paramètre CDTREVIS en nomenclature 098 vaut 0 ou NULL, alors le taux final ne doit pas être renseigné mais uniquement la marge (a valeur du taux final est alors calculée par addition entre la valeur du taux de base et la marge)

Si le code du taux de base n'est pas renseigné, alors ni la marge ni le taux final ne doivent être renseignés. Seul le taux fixe doit être alimenté.

Nom : margin
Description : Marge.
Commentaire :

La marge ne doit être renseignée que si le taux de base est renseigné pour la révison et que la marge est autorisée (Montant 4 du paramètre CDTREVIS en nomenclature 098 différent de 1).

Si la marge peut être renseignée mais qu’elle ne l’est pas, alors elle est affectée à la valeur  de la condition VTBPRG.

Si le dossier est avec marge variable et type d'intérêts révisables alors la marge est forcée à 0.

La marge doit être comprise dans les bornes minimum et maximum paramétrées dans la condition VTBPRG.

Le taux final est alors calculé automatiquement. Et il doit être :

  • Supérieur ou égal au taux de révision minimum,
  • Inférieur ou égal au taux de révision maximum.

Commentaire :

La marge est alors calculée automatiquement et doit être comprise dans les bornes minimum et maximum paramétrées dans la condition VTBPRG.

Nom : fixRate
Description : Taux fixe.
Commentaire :

Pour la révison, soit le code du taux de base, soit le taux fixe doit être renseigné.

S’il est renseigné, le taux fixe doit être compris dans les bornes minimum et maximum paramétrées dans la condition INssss (ssss = source de financement) ou  INTttt (ttt = type de prêt) ou INTPRT.

Nom : baseRateCode
Description : Code du taux de base.
Commentaire :

Le code du taux de base ne doit être renseigné que dans le cas d'un taux indexé et dans le cas où la gestion des codes de taux de base ne se fait pas par le biais de conditions (le Libellé 2 du paramètre CDTPAR2 doit être égal à O).

Le code du taux de base renseigné doit :

  • exister dans la table des taux de base,
  • être autorisé pour le type de prêt et la devise du prêt.

Si aucun code de taux de base n'est renseigné en entrée, alors le code du taux de base paramétré pour le type et la devise du prêt est affecté par défaut s’il est unique, sinon la saisie du taux de base est obligatoire.

Nom : marginValue
Description : Valeur de la marge.
Commentaire :

La valeur de la marge peut être renseignée uniquement si elle est saisissable (Libellé 2 du paramètre PRET en nomenclature 098 égal à O). Si le taux est indexé, alors le taux d’intérêts final est automatiquement recalculé en fonction de la marge renseignée et du taux de base recherché à partir du code de taux de base lu dans la condition TBssss (ssss = source de financement) ou à défaut TBPRET.

Si la valeur de la marge ne peut pas être renseignée (Libellé 2 du paramètre PRET en nomenclature 098 égal à N), alors elle est positionnée à la valeur de la condition VBssss (ssss = source de financement) ou à défaut INTPRI.
