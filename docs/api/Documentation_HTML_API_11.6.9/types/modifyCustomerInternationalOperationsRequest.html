  • modifyCustomerInternationalOperationsRequest
Nom : modifyCustomerInternationalOperationsRequest
Description : Requête de modification des opérations à l'étranger d'un client.
Choix : Non
  • modifyCustomerInternationalOperationsRequest
    • customerCode : customerCode
    • [M] internationalOperation : customerInternationalOperationInformations
      • customerInternationalOperationType : customerInternationalOperationType
      • customerInternationalOperationCode : charMax3

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : customerCode
Description : Code du client.
Commentaire :

Le code client n’est pas obligatoire si le service de modification des opérations à l’étranger d’un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : internationalOperation
Description : Opération à l'étranger pour le client.
Limite du nombre d'occurences mini-maxi : aucune aucune
Commentaire :

L’indicateur d’opérations à l’étranger doit être positionné à O pour le client.

Si l’indicateur d’opérations à l’étranger est positionné à O pour le client, alors au moins une opération à l’étranger doit être renseignée.

Le nombre maximum d’opérations à l’étranger autorisées pour un client est de 198 (99 de type "pays" + 99 de type "devise").

Nom : customerInternationalOperationInformations
Description : Informations d'une opération à l'étranger d'un client.
Nom : customerInternationalOperationType
Description : Type de l'opération.
Nom : customerInternationalOperationType
Description : Type de lien.
Format : string
Valeurs possibles :
P Pays
D Devise
Nom : customerInternationalOperationCode
Description : Code de l'opération.
Commentaire :

Le code de l’opération correspond au code pays si le type de l’opération est Pays (P). Dans ce cas, il doit exister en nomenclature 040.

Le code de l’opération correspond au code devise si le type de l’opération est Devise (D). Dans ce cas, il doit exister en nomenclature 002 pour une devise alphanumérique ou en nomenclature 005 pour une devise numérique.

Valeurs par défaut :

  • opetr_pays si le code de l’opération est Pays (P)
  • opetr_dev si le code de l’opération est Devise (D)
Nom : charMax3
Description : Chaîne composée d'au plus 3 caractères.
Format : string
Longueur mini-maxi : 1 - 3
