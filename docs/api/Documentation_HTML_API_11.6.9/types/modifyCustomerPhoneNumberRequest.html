Nom : customerPhoneNumber

Nom : createCustomerPhoneNumberRequest
Description : Requête de création d'un numéro de téléphone d'un client.
Nom : identifier
Description : Identifiant du numéro de téléphone.
Commentaire :

Un contrôle de saisie obligatoire d’au moins un numéro de téléphone pour le client est effectué via la valeur par défaut telfmt.

Nom : phoneNumberCreateIdentifier
Description : Identifiant d'un numéro de téléphone d'un client.
Nom : customerCode
Description : Code du client.
Commentaire :

Le code client n’est pas obligatoire si le service de création d’un numéro de téléphone pour un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

S’il est renseigné, il doit correspondre à un client existant dans la table des clients.

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : type
Description : Type de téléphone.
Commentaire :

Le code du type de téléphone doit exister en nomenclature 182.

Nom : char3
Description : Type avec un nombre de caractères égal à 3.
Format : string
Longueur fixe : 3
Nom : phoneNumber
Description : Numéro de téléphone.
Commentaire :

La valeur du numéro de téléphone est contrôlée à partir de son format s’il est valorisé.

Le numéro de téléphone est alors complété avec l’indicatif national paramétré en nomenclature 199 dans le Libellé 3 en appliquant le masque paramétré dans le Libellé 4 (i représentant l’indicatif et n un chiffre du numéro de téléphone) puis amputé du premier zéro du numéro (situé juste après l’indicatif national) si le Taux 1 vaut 1.

La longueur du numéro de téléphone après l’ajout de l’indicatif national et la suppression du premier zéro éventuel doit correspondre à la longueur paramétrée dans le Montant 1.

Nom : charMax20
Description : Chaîne composée d'au plus 20 caractères.
Format : string
Longueur mini-maxi : 1 - 20
Nom : format
Description : Format du numéro de téléphone.
Commentaire :

S'il est valorisé, le format du numéro de téléphone doit exister en nomenclature 199.

Nom : char3
Description : Type avec un nombre de caractères égal à 3.
Format : string
Longueur fixe : 3
