  • modifyCustomerProfessionsAndIncomesRequest
Nom : modifyCustomerProfessionsAndIncomesRequest
Description : Requête de modification des professions et revenus d'un client.
Choix : Non
  • modifyCustomerProfessionsAndIncomesRequest
    • customerCode : customerCode
    • [M] professionAndIncomes : customerProfessionAndIncomesInformations
      • hireDate : date
      • professionCode : professionCode
      • employerCode : employerCode
      • incomesBracketCode : incomesBracketCode
      • employerDepartment : charMax30

Légende :

  • = Elément obligatoire en saisie.
  • [M] = Elément multiple (0 à n occurrences).
  • [C] = Elément composé d'un choix à effectuer. Un seul choix autorisé.

Remarque : Un même élément peut regrouper plusieurs caractéristiques

Nom : customerCode
Description : Code du client.
Commentaire :

Le code client n’est pas obligatoire si le service de modification des professions et revenus d’un client est appelé par le service createCustomer, et est obligatoire dans tous les autres cas.

Si le contrôle sur le contentieux est demandé dans le Montant 1 (égal à 1) du paramètre AWMH dans la nomenclature 098 ou si le paramètre AWMH n’existe pas dans la nomenclature 098, alors le client ne doit pas être présent dans la table des clients en contentieux.

Nom : customerCode
Description : Code client.
Format : string
Longueur mini-maxi : 1 - 15
Nom : professionAndIncomes
Description : Informations d'une profession et des revenus du client.
Limite du nombre d'occurences mini-maxi : aucune aucune
Commentaire :

Les professions et revenus doivent être classés par date d’embauche décroissante.

Le nombre maximum de professions et revenus autorisés pour un client est de 99.

Nom : customerProfessionAndIncomesInformations
Description : Informations d'une profession et des revenus d'un client.
Nom : hireDate
Description : Date d'embauche.
Commentaire :

Si un nombre de jours est paramétré dans le Montant 1 de la nomenclature 098 pour le paramètre CONTRDEMB, alors la date d’embauche doit être inférieure ou égale à la date comptable à laquelle on ajoute ce nombre de jours.

Si un nombre de jours est paramétré dans le Montant 2 de la nomenclature 098 pour le paramètre CONTRDEMB et si le client est un client collectif (Montant 2 renseigné à 2 dans la nomenclature 036 pour le code intitulé du client), alors la date d’embauche doit être supérieure ou égale à la date comptable à laquelle on retire ce nombre de jours.

La date d’embauche doit être supérieure à la date de naissance du client.

Nom : date
